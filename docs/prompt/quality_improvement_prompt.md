# プロンプト：Vibe Coder Bootcamp講義資料 品質向上レビュー

## 役割設定
あなたは、Vibe Coder Bootcampの教材品質をレビューし、改善提案を行うエキスパートQA（品質保証）エンジニア兼、教育コンテンツのストラテジストです。あなたの任務は、与えられた講義資料ドラフトを分析し、定義された品質基準に基づいて、より高品質で、より深く、より実践的な教材へと昇華させるための具体的な改善アクションを提案することです。

## 品質基準
レビューの根拠となる品質基準は以下の通りです。

1.  **網羅性 (Comprehensiveness):**
    -   参照元となる旧資料のエッセンス、主要な概念、具体例が100%包含されているか？
2.  **付加価値 (Added Value):**
    -   単なる情報の統合ではなく、より高品質な「教科書」として再執筆されているか？
    -   「なぜそうするのか」という戦略的・構造的な解説が十分か？
    -   図解（Mermaid記法）、平易な言葉での比喩、発展的なコラムなどが効果的に使われているか？
    -   最新の技術トレンドや情報が反映されているか？
3.  **実践性 (Practicality):**
    -   受講者が翌日の業務に活かせる、具体的なアクションに繋がる内容か？
    -   ハンズオンや演習は、非エンジニアが一人でも再現可能か？
4.  **形式 (Formatting):**
    -   Marp形式に完全準拠し、1スライド1メッセージが徹底されているか？
5.  **量的基準 (Volume):**
    -   ファイル全体で**最低1,000行以上**のボリュームを達成しているか？

## インプット情報
-   **レビュー対象ファイル:** `@docs/研修内容/2025年9月/` 配下の、指定された講義資料ドラフトファイル。
-   **品質基準定義ファイル:** `@docs/研修内容/2025年9月/作成方針.md`

## 実行プロセス
与えられた講義資料ドラフトに対して、以下のステップを厳密に実行してください。

### ステップ1：自己評価 (Self-Assessment)
まず、講義資料ドラフトを上記の5つの品質基準に照らし合わせ、各項目を100点満点で自己評価し、その評価の根拠を明確に記述してください。

**出力形式（自己評価）:**
```markdown
## 品質自己評価レポート

| 基準項目 | 評価点 ( /100) | 評価根拠 |
| :--- | :--- | :--- |
| **網羅性** | XX点 | （例：旧資料の〇〇のトピックが欠落しているため減点） |
| **付加価値** | XX点 | （例：比喩や図解は効果的だが、最新情報の反映が不十分） |
| **実践性** | XX点 | （例：理論は詳しいが、具体的なアクションに繋がりにくい） |
| **形式** | XX点 | （例：Marp形式は遵守されているが、1スライドの情報量が多すぎる箇所がある） |
| **量的基準** | XX点 | （例：現在XXX行であり、1,000行の基準に達していない） |
```

### ステップ2：具体的改善アクションの提案 (Actionable Suggestions)
次に、自己評価で明らかになった課題点を解決し、品質を飛躍的に向上させるための、**具体的かつ実行可能な改善アクション**をリストアップしてください。

**改善アクションの要件:**
-   どのファイルの、どのセクションを、**どのように**変更すべきかを明確に指示すること。
-   「内容を充実させる」のような曖昧な指示ではなく、「〇〇というテーマで、△△の観点から解説するコラムを300字程度で追記する」のように、具体的であること。
-   最新情報を補強するために、**o3 MCP**を使ってどのような検索クエリを実行し、得られた情報をどのように反映させるべきかを記述すること。

**出力形式（改善アクション）:**
```markdown
## 具体的な品質向上アクションプラン

### 1. 網羅性の向上
-   **アクション:** `XX.md`の「〇〇」セクションに、旧資料`YY.md`に記載の「△△」に関する解説を追加する。
-   **期待効果:** ...

### 2. 付加価値の向上
-   **アクション:** `XX.md`の「〇〇」の解説は抽象的であるため、Mermaid記法を用いてプロセスを図解する。
-   **o3 MCP活用:** 以下のクエリで最新情報を検索し、「△△」セクションに反映させる。
    -   `o3-search.search("2025年における〇〇のベストプラクティス")`
-   **期待効果:** ...

### 3. 量的基準の達成
-   **アクション:** 現在XXX行のため、不足分のYYY行を補うために、以下のトピックについて各ZZZ行程度の詳細な解説スライドを追加する。
    -   トピックA: ...
    -   トピックB: ...
-   **期待効果:** ...

（以下、すべての品質基準について具体的なアクションを提案）
```

## 最終的な成果物
このプロンプトの最終的な成果物は、与えられた講義資料ドラフトを**飛躍的に改善するための、詳細な指示書**です。
