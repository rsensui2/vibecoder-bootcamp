# デモデイ発表資料生成プロンプト - 使用方法ガイド

このディレクトリには、VibeCoder育成プログラムのデモデイで使用する発表資料を自動生成するためのプロンプトが含まれています。

## 📁 ファイル構成

```
docs/prompt/demoday/
├── demoday_presentation_generator.md  # メインプロンプト
└── README.md                          # このファイル（使用方法ガイド）
```

## 🎯 目的

このプロンプトは、企業向けVibeCoder育成プログラムのデモデイで使用する発表資料を、企業ごとにカスタマイズして生成するために設計されています。

### 対象者
- **審査員**: 初めてこの講義のことを知る人（経営層、人事担当者など）
- **受講生**: プログラムを修了した参加者
- **関係者**: 今後の導入を検討する企業担当者

## 📝 使用方法

### ステップ1: 必要な情報を準備

以下の情報を事前に準備してください：

#### 必須情報
1. **基本情報**
   - 提供先企業名
   - 受講生人数
   - 開催期間（開始日〜終了日）
   - デモデイ日時

2. **満足度データ（アンケート結果）**
   - 全体満足度の平均値（小数点第2位まで）
   - 面白さの平均値（小数点第2位まで）
   - 難易度の平均値（小数点第2位まで）
   - 回答者数と総受講生数

3. **受講生の声**
   - 代表的なコメント3-5件

4. **代表的な成果物**
   - 受講生が作成したアプリケーション3件程度（名前と簡単な説明）

#### オプション情報
5. **講師所感**（任意）
   - 手応え
   - 反省点と課題
   - ※入力しない場合は、デフォルトテンプレートが使用されます

### ステップ2: プロンプトに情報を入力

`demoday_presentation_generator.md` を開き、**【入力情報】** セクションの各項目に準備した情報を記入します。

#### 入力例

```markdown
### 基本情報
- **提供先企業名**: 武蔵野EMC株式会社
- **受講生人数**: 8名
- **開催期間**: 2025/10/01 〜 2025/10/29
- **デモデイ日時**: 2025年10月29日(火) 14:00-16:00

### 満足度データ（アンケート結果の平均値）
- **全体満足度**: 4.55 / 5.0
- **面白さ**: 4.50 / 5.0
- **難易度**: 3.90 / 5.0
- **回答者数**: 8名 / 8名

### 受講生の声（代表的なコメント3-5件）
1. "AIを使ってやる、むしろそうすべきだと感じた"
2. "ビズ側の人材が出来るようになった時のインパクトが凄過ぎる"
3. "他人の詰まりポイントから学べた"

### 代表的な成果物（3件程度）
1. **タスク管理アプリ**: チーム内のタスクを共有・管理できるWebアプリケーション
2. **社内掲示板**: 部署間の情報共有を促進するSNS型アプリケーション
3. **在庫管理システム**: リアルタイムで在庫状況を確認できる業務効率化ツール
```

### ステップ3: AIに生成を依頼

プロンプト全体をClaude、ChatGPT、Cursorなどのチャット型AIにコピー＆ペーストして、発表資料の生成を依頼します。

#### 推奨プロンプト
```
このプロンプトに従って、デモデイ発表資料を生成してください。
入力情報と固定情報を使用し、出力指示に従ってMarkdown形式で出力してください。
```

### ステップ4: 生成された資料を確認・調整

AIが生成した資料を確認し、必要に応じて以下を調整します：

- 企業名や日時などの可変情報が正しく反映されているか
- 満足度データが正確に記載されているか
- 受講生の声が適切に引用されているか
- Mermaid図が正しく表示されるか
- 講師所感が適切に挿入されているか

### ステップ5: スライド化（オプション）

生成されたMarkdownファイルを、reveal.jsやMarpを使ってスライド形式に変換できます。

#### Marpの場合
```bash
# Marpをインストール（未インストールの場合）
npm install -g @marp-team/marp-cli

# HTMLに変換
marp demoday_presentation_武蔵野EMC.md -o demoday_presentation_武蔵野EMC.html

# PDFに変換
marp demoday_presentation_武蔵野EMC.md -o demoday_presentation_武蔵野EMC.pdf --pdf
```

#### reveal.jsの場合
- 生成されたMarkdownをreveal.jsのテンプレートに貼り付けて使用

## ⚙️ カスタマイズのポイント

### 講師所感について
- **入力した場合**: 入力内容がそのまま使用されます
- **入力しない場合**: デフォルトテンプレートが自動で挿入されます

### 受講生の声
- **必須**: 最低3件
- **推奨**: 3-5件
- **形式**: 引用符で囲んだ実際のコメント

### 成果物
- **推奨数**: 3件程度
- **記載内容**: アプリ名と1-2文の簡単な説明

## 🔧 トラブルシューティング

### Mermaid図が表示されない
- Markdownプレビューツールによっては、Mermaid図がサポートされていない場合があります
- VS Code、Obsidian、Typora、GitHubなどMermaid対応エディタを使用してください

### 満足度データの計算方法
- 未回答者は除外して平均値を算出
- 小数点第2位まで表示（例: 4.55）
- 5点満点で評価

### プロンプトが長すぎてエラーになる
- Claude、GPT-4などのコンテキスト長が長いモデルを使用してください
- 必要に応じて、入力情報を分割して生成することも可能です

## 📌 注意事項

1. **企業名の取り扱い**
   - 提供先企業名は正確に記載してください
   - ファイル名にも企業名が含まれます

2. **満足度データの正確性**
   - アンケート結果は正確に入力してください
   - 平均値の計算は未回答者を除いて行ってください

3. **著作権と機密情報**
   - 受講生の声やコメントは、事前に掲載許可を得てください
   - 企業の機密情報が含まれないように注意してください

4. **審査用アプリのURL**
   - 現在は `https://demodaybasic.vercel.app/` で固定
   - 変更が必要な場合は、プロンプトの【固定情報】セクションを編集してください

## 💡 ヒント

- **初回使用時**: サンプルデータで一度生成してみて、出力形式を確認することをお勧めします
- **複数企業対応**: 入力情報だけを変更すれば、複数企業向けの資料を簡単に作成できます
