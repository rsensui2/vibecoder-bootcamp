# **第二回**
## 開発環境構築とエラー対処法

---

### 本日のアジェンダ
1.  **ターミナルとの対話 (15分)**
    -   黒い画面の正体と、基本的なコマンド
    -   MacとWindowsの違い
2.  **Next.js環境構築 (15分)**
    -   `npm install` と `npm run dev` の意味
    -   ローカルサーバーを立ち上げる
3.  **エラーとの付き合い方 (15分)**
    -   よくあるエラーとその原因
    -   Chrome開発者モードの使い方
    -   エラーはAIへの最高の質問文

---

### 1. ターミナルとの対話
#### 「黒い画面」は怖くない
-   **（ターミナル（Mac）/ PowerShell（Windows）が、PCに直接命令を出すための「対話ウィンドウ」であることを、比喩を使って解説）**
-   **（なぜGUIだけでなくCUI（コマンド）を使うのかを、「定型作業の自動化」「AIとの連携のしやすさ」の観点から説明）**

---

#### 覚えておくべき基本コマンド
-   **（`cd`コマンドについて、フォルダを「移動する」という概念を、図解を交えて解説）**
-   **（o3 MCPを活用し、「Mac ターミナル cd 使い方」「Windows PowerShell cd 使い方」などで最新の解説記事を検索し、その内容を基に、OSごとの違い（パスの区切り文字など）を丁寧に解説）**
-   **（`ls` (Mac) / `dir` (Windows) コマンドも紹介）**

---

### 2. Next.js環境構築
#### アプリケーションの「原材料」を揃える
-   **（`npm install`コマンドが、`package.json`という「レシピ本」を読んで、必要な「原材料（ライブラリ）」をインターネットからダウンロードしてくる作業であることを解説）**
-   **（`node_modules`フォルダが、その原材料が保管される「巨大な冷蔵庫」であることを説明）**

---

#### アプリケーションを「試作」する
-   **（`npm run dev`コマンドが、ローカルPC上に「試作専用の厨房（開発サーバー）」を立ち上げ、いつでも味見（動作確認）できる状態にするコマンドであることを解説）**

---

### 3. エラーとの付き合い方
#### よくあるエラー1：「ポートが既に使用されています」
-   **（`localhost:3000`が既に埋まっているエラーについて、「厨房のコンロが1つしかないのに、2つの料理を同時に作ろうとしている」状態だと比喩で解説）**
-   **（解決策1：別のコンロ（ポート）を使う方法と、解決策2：前の料理を片付ける（プロセスを停止する）方法を、Cursor Agentへの具体的な指示プロンプト例と共に提示）**

---

#### エラーの見分け方：厨房のエラーか、ホールからのクレームか
-   **（ターミナルに表示されるエラーが「厨房（バックエンドやビルド）」の問題であり、ブラウザの画面が崩れたり、何も表示されなかったりするのが「ホール（フロントエンド）」の問題であると解説）**

---

#### Chrome開発者モード：お客様の声を聞く
-   **（フロントエンドのエラーを確認するために、Chrome開発者モードの「Console」タブが「お客様からのクレームや要望がリアルタイムで届く場所」であることを解説）**
-   **（開き方（右クリック→検証）と、赤いエラーメッセージの見方を、スクリーンショットを挿入する前提で丁寧に説明）**

---

#### エラーはAIへの最高の質問文
-   **（エラーメッセージを恐れるのではなく、「AIへの最高の質問文」として捉えるマインドセットを強調）**
-   **（ターミナルやChrome ConsoleのエラーメッセージをそのままCursorにコピー＆ペーストし、「このエラーの原因と解決策を教えて」と質問するだけで、ほとんどの問題が解決することを実演形式で解説）**

---
