#### フロントエンド：ユーザーとの対話担当
-   **役割:** ユーザーが直接見て、触れるすべての部分を担当します。
-   **責任範囲（用語整理）:**
    -   **UI (User Interface):** 見た目のデザイン、ボタンやフォームの配置。
    -   **UX (User Experience):** 使いやすさ、心地よさ、楽しさといった体験全体。
    -   **インタラクション:** ボタンをクリックした時の反応、アニメーションなど。
-   **使用技術:** HTML (骨格), CSS (デザイン), JavaScript (動き)

---

#### バックエンド：裏方のデータ処理・管理担当
-   **役割:** ユーザーの目には見えない裏側で、重要なデータを安全に処理・保管します。
-   **責任範囲（用語整理）:**

    -   **データ処理:** データベースへのデータの保存、更新、削除。
    -   **セキュリティ:** 不正なアクセスからデータを守る。
-   **使用技術:** Node.js, Python, Go, Javaなど

### API：外部サービスの力を借りる魔法
#### APIとは？ - 機能の「コンセント」
API (Application Programming Interface) とは、他のサービスが提供する機能を使うための**「コンセント」**のようなものです。

-   コンセントに掃除機を繋げば、電気という力を借りて部屋をきれいにできます。
-   同様に、APIを使えば、GoogleやOpenAIが提供する高度な機能を、自分のアプリに組み込むことができます。

---

#### 具体的なAPIの利用シーン
-   **Google Maps API:**
    -   **借りたい力:** 地図表示、経路検索
    -   **あなたのアプリでできること:** 店舗情報ページに地図を埋め込む、配達ルートを計算する。
-   **OpenAI / Claude API:**
    -   **借りたい力:** 高度な文章生成、要約、翻訳
    -   **あなたのアプリでできること:** AIチャットボット機能、ブログ記事の自動生成機能。
-   **Stripe API:**
    -   **借りたい力:** クレジットカード決済
    -   **あなたのアプリでできること:** 商品やサービスのオンライン販売。
-   **Clerk API (今後の講義で扱います):**
    -   **借りたい力:** 高度で安全なログイン機能
    -   **あなたのアプリでできること:** GoogleやGitHubアカウントでのログイン機能を数行のコードで実装。
-   **Supabase API (今後の講義で扱います):**
    -   **借りたい力:** データベースの安全な操作
    -   **あなたのアプリでできること:** プログラムから安全にデータを読み書きする。

### APIキー：魔法を使うための「鍵」
#### APIキーとは？ - コンセントを使うための「認証キー」
APIキーは、その「コンセント（API）」を使うことを許可された、あなた専用の**「認証キー」**です。

-   **役割:**
    -   **認証:** あなたが正当な契約者であることをサービス提供者に証明する。
    -   **利用量計測:** あなたがどれだけサービス（電力）を使ったかを計測し、料金を請求する。
-   **重要性:** APIキーは、あなたのアカウントに紐づく非常に重要な秘密情報です。**絶対に他人に知られてはいけません。**

---

#### .envファイルでの安全な管理
APIキーのような秘密情報は、`.env.local`という特別なファイルに記述します。このファイルは、GitHubにアップロードされない設定になっているため、安全にキーを管理できます。

**記述例 (`.env.local`):**
```
# Google Maps API Key
NEXT_PUBLIC_GOOGLE_MAPS_API_KEY="AIzaSy...Your...Key"

# OpenAI API Key
OPENAI_API_KEY="sk-proj-...Your...Key"
```
-   **`NEXT_PUBLIC_`:** 接頭辞がついているキーは、フロントエンド（ブラウザ）からでも安全に使える公開キーです。
-   **接頭辞なし:** バックエンド（サーバー）でのみ使用する、より厳重に管理すべき秘密キーです。
