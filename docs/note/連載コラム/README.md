## 🚀 クイックスタート

### 1. 全体構成を確認
まず [00_MASTER_OUTLINE.md](./00_MASTER_OUTLINE.md) を読んで、全23本のコラムの全体像を把握してください。

### 2. コラムを生成する
生成したいコラムのプロンプトファイル（例：`prompts/section1_認識を揺さぶる/01_プログラミングを学ぶな_prompt.md`）を開き、最下部の**「実際の生成プロンプト」**セクションをClaude/ChatGPTにコピー＆ペーストします。

### 3. 講義資料を読み込ませる
プロンプトに記載されている「参照すべき講義資料」を読み込ませます。

#### 方法A: ファイルパスを指定（Claude Code推奨）
```
以下のファイルを参照してください：
/Users/sensuiryousuke/.../docs/研修内容/2025年11月/第一回/1-1_オリエンテーションとマインドセット変革.md
```

#### 方法B: 内容を直接貼り付け（ChatGPT等）
該当ファイルの内容をコピーしてプロンプトに追加します。

### 4. 生成されたコラムを確認・調整
生成されたコラムを `generated/` フォルダに保存し、必要に応じて微調整します。

## 📝 各プロンプトファイルの構造

各プロンプトファイルは以下のセクションで構成されています：

### 1. メタ情報
- コラム番号、セクション、文字数目標、出典講義、参照資料

### 2. コンテクスト設定
- このコラムの役割
- 前後のコラムとの関係
- ターゲット読者の状態

### 3. 核心メッセージ
- このコラムで伝えるべき本質（1文）
- 読後に読者に残すべき3つのポイント

### 4. 構成指示
- タイトル案
- 導入部・本論部・実践パート・結論部の詳細指示
- 必須で含める具体例・メタファー
- 知的好奇心を刺激するポイント

### 5. トーン＆スタイル指示
- 文体、使うべきレトリック、避けるべき表現

### 6. 参照すべき講義資料の該当箇所
- ファイルパスとセクション名

### 7. 生成時の注意事項
- 必ず守るべきルール
- 柔軟に調整可能な部分

### 8. 品質チェックリスト
- 生成後に確認すべき項目

### 9. 実際の生成プロンプト
- Claude/ChatGPTに直接コピペできる形式

## 🎯 効果的な使い方

### パターン1: 順番通りに生成（推奨）
**#1 → #2 → #3 → ...** の順で生成することで、各コラム間の自然な流れが保たれます。

### パターン2: セクション単位で生成
例えば「セクション1（#1-4）を一気に書く」など、テーマごとに集中して作業できます。

### パターン3: 興味のあるコラムから生成
各コラムは独立性も持たせているため、順不同でも問題ありません（ただし、誘導文は前後関係を想定しています）。

## ⚙️ カスタマイズのヒント

### 文字数を調整したい場合
プロンプトの「文字数目標」セクションを変更し、各部の文字数配分も調整してください。

### トーンを変えたい場合
「トーン＆スタイル指示」セクションを編集します。泉水様の既存Note記事（参照資料に記載）を追加で読み込ませると、より文体が近づきます。

### 具体例を追加したい場合
「構成指示」→「必須で含める具体例・メタファー」に追加します。講義資料の該当箇所も明示してください。

## 🔧 トラブルシューティング

### Q: 生成されたコラムが文字数オーバー/アンダーになる
**A**: プロンプトの「文字数目標」を明示的に強調し、「厳密に○○字以内」と指示してください。

### Q: トーンが硬すぎる/柔らかすぎる
**A**: 既存Note記事（https://note.com/rsensui/n/n5aa6e6289242 または https://note.com/rsensui/n/n41598ed615ae）を追加で読み込ませ、「この文体を模倣してください」と指示します。

### Q: 専門用語が多すぎる
**A**: プロンプトに「ターゲット読者は非エンジニア」「専門用語は必ず日常的な比喩で説明」と追記します。

### Q: 次のコラムへの誘導が不自然
**A**: 前後のコラムを同じAIセッションで連続生成すると、流れが自然になります。

## 🎨 スタイルガイド

### 見出しレベル
- H1 (`#`): タイトルのみ
- H2 (`##`): 主要セクション
- H3 (`###`): サブセクション

### コードブロック
実装例やコマンド例には必ずコードブロックを使用：
\```typescript
// 例
const example = "Hello World";
\```

### 引用
講義資料や既存記事からの引用には `>` を使用：
> これは引用例です

### リスト
- 箇条書きは `-` を使用
- 番号付きリストは `1.` を使用

### 強調
- **太字**: 重要な概念やキーワード
- *イタリック*: 用語の初出
- `コード`: コマンド、変数名、ファイル名

## 💡 Tips

### Tip 1: バッチ生成
複数のコラムを一度に生成したい場合、Claude Codeのようなファイル横断ツールを使うと効率的です。

### Tip 2: ブランディング一貫性
全てのコラムで以下の要素を統一：
- Vibe Coder / vibeCoding（表記ゆれ注意）
- Context Control Technology
- vibecoderbootcamp.com への誘導

### Tip 3: 読者フィードバック
実際に公開後、読者の反応を見て、後続のコラムのトーンや内容を微調整できます。

### Tip 4: セクション間の休止
各セクションの最後のコラム（#4, #10, #16, #20）は総まとめ的な内容なので、次のセクションに入る前に1-2日空けると読者の理解が深まります。
