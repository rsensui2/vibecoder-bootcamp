## 2. コンテクスト設定

### 2.1 このコラムの役割
- **前回（#5）からの流れ**: 前回、ChatGPTのようなチャット型AI（Level 1-2）では単発の会話文脈しか扱えないのに対し、CursorのようなIDE型AI（Level 3）はプロジェクト全体のコンテクストを理解できることを説明した
- **今回の焦点**: CursorのComposer Modeという、複数ファイルを同時に編集する革新的な機能を深掘りし、「一文字でも間違えればエラー」というプログラミングの厳しさにどう対応するかを示す
- **次回（#7）への橋渡し**: Composer Modeでの効率的なコンテクスト制御には、音声入力による高速な言語化が不可欠であることを示唆し、次回の「Text is KING - 音声入力がもたらす革命」へつなげる

### 2.2 読者の現在地
- Level 3のコンテクスト（プロジェクト全体）の重要性は理解した
- しかし「実際にどうやってプロジェクト全体を制御するのか？」という具体的な方法がまだ見えていない
- ChatGPTでは「1つのファイルをコピペ→修正→また別のファイルをコピペ→修正」という非効率な作業を繰り返していた経験がある
- 「複数ファイルを同時に編集」というイメージが湧いていない

### 2.3 このコラムで達成すべきこと
- Composer Modeの「複数ファイル同時編集」という革新性を体感させる
- 「一文字でも間違えればエラー」という厳しさに対して、Composer Modeがどう整合性を保つかを理解させる
- Context Control Technology（言語化力 + AI使いこなし + ドキュメント管理）の「AI使いこなし」部分の具体例を示す
- Cursorを使いたくなる気持ちを強化する

## 4. 読後に読者に残すべき3つのポイント

1. **Composer Modeの本質**: 単なる「複数ファイル編集」ではなく、「プロジェクト全体の整合性を保ちながら同時編集」する能力
2. **整合性の自動維持**: 人間が手動で管理すると「一文字のミス」で崩壊するプロジェクト全体の整合性を、Composer ModeはAIが自動で維持する
3. **Context Control訓練**: この厳しい整合性要求こそが、Context Control Technology（180万円価値）を鍛える最高の訓練環境である

### 5.2 導入部（400-500字）
**目的**: 読者が普段経験している「ファイル間の不整合地獄」を想起させ、問題意識を喚起する

**構成要素**:
- **Question**: 「あなたは今まで、1つのファイルを修正したら、関連する5つのファイルも修正しなければならず、しかもその5つのファイルすべてで整合性を保たなければならない、という地獄を経験したことはありますか？」
- **具体例**:
  - ToDo管理アプリで「優先度フィルター機能」を追加する場合
  - 修正が必要なファイル: `components/TodoList.tsx`, `types/todo.ts`, `lib/api.ts`, `components/FilterBar.tsx`, `pages/index.tsx`（5ファイル）
  - ChatGPTでの作業: 1つずつコピペ→修正指示→コピペ→次のファイル→コピペ...（5回繰り返し）
  - **問題**: 4つ目のファイルを修正中に、1つ目のファイルとの整合性が崩れることに気づく→また1つ目に戻る→無限ループ
- **痛み**: 「一文字でも間違えればエラー」なのに、人間が手動で5ファイルの整合性を保つのは不可能
- **導入の締め**: 「Composer Modeは、この地獄を終わらせます。」

### 5.3 本論部1: Composer Modeとは何か（600-700字）
**目的**: Composer Modeの機能を正確に説明する

**構成要素**:
- **定義**: Cursorの機能の1つで、複数ファイルを同時に選択し、プロジェクト全体のコンテクストを理解した上で、整合性を保ちながら同時編集する機能
- **従来の方法との比較**:
  - **ChatGPT**: 1ファイルずつコピペ→修正→次のファイル（Level 1-2のコンテクスト）
  - **Cursor Chat**: 1ファイルをその場で編集（Level 3だが単一ファイル）
  - **Composer Mode**: 複数ファイルを同時に、プロジェクト全体の整合性を保ちながら編集（Level 3 + Multi-File）
- **技術的背景**（講義資料「2-3 Cursor環境構築」から引用）:
  - Codebase Indexingにより、プロジェクト全体を理解
  - Multi-Agentsにより、複数ファイルを並行処理
  - 整合性チェック機能により、型定義・関数シグネチャ・import文などを自動で一致させる
- **メタファー**: 「オーケストラの指揮者」
  - ChatGPT = 1人の演奏者に1つずつ指示を出す（バラバラに演奏）
  - Composer Mode = オーケストラ全体を見渡し、すべての演奏者が調和するよう指揮する

### 5.4 本論部2: 整合性維持の具体例（700-800字）
**目的**: 「一文字でも間違えればエラー」という厳しさに対して、Composer Modeがどう対応するかを示す

**構成要素**:
- **具体例**: ToDo管理アプリに「優先度フィルター機能」を追加する場合
- **Composer Modeでの作業フロー**:
  1. Composer Modeを起動（Cmd+I または Ctrl+I）
  2. 関連する5ファイルを選択（または自動検出）
  3. 1つの指示を出す: 「優先度フィルター機能を追加してください。高・中・低の3段階で、ボタンクリックでフィルタリングできるようにしてください」
  4. AIがプロジェクト全体を理解した上で、5ファイルを同時に編集
  5. 型定義、関数シグネチャ、import文、コンポーネント間のプロパティ受け渡し、すべてが自動で整合性を保たれる
- **整合性の例**（具体的なコード例を含める）:
  - `types/todo.ts`に`priority: 'high' | 'medium' | 'low'`を追加
  - → `components/TodoList.tsx`の型定義が自動で更新
  - → `lib/api.ts`のフィルタリング関数が自動で追加
  - → `components/FilterBar.tsx`の新規コンポーネントが自動で作成
  - → `pages/index.tsx`で`FilterBar`が自動でimport & 配置
- **人間が手動でやった場合のミス例**:
  - `types/todo.ts`で`priority: 'high' | 'medium' | 'low'`と定義
  - でも`components/TodoList.tsx`で`priority: 'High' | 'Medium' | 'Low'`（大文字）と書いてしまう
  - → TypeScriptエラー発生
  - → 「一文字でも間違えればエラー」の典型例
- **Composer Modeの威力**: こうしたミスを一切起こさない整合性維持

### 5.5 本論部3: Context Control訓練としての価値（500-600字）
**目的**: プログラミング（★★★★★難易度）をContext Control訓練として選ぶ理由を再強調

**構成要素**:
- **180万円の価値**（武蔵野大学EMC企業向けプログラムから引用）:
  - 目標: Context Control Technology = 言語化力 + AI使いこなし + ドキュメント管理
  - 訓練素材: プログラミング（★★★★★）
  - 理由: 「一文字でも間違えればエラー」という厳しさが、言語化力を極限まで鍛える
- **Composer Modeとの関係**:
  - プロジェクト全体の整合性を保つには、「何を実現したいか」を正確に言語化する必要がある
  - 曖昧な指示: 「フィルター機能を追加して」→ AIは迷う
  - 正確な指示: 「優先度フィルター機能を追加してください。高・中・低の3段階で、ボタンクリックでフィルタリングできるようにしてください」→ AIは正確に実装
  - この「正確な言語化」こそが、Context Control Technologyの核心
- **他のタスクとの比較**:
  - メール文章（★☆☆☆☆）: 多少曖昧でも伝わる
  - プレゼン資料（★★☆☆☆）: レイアウトが少し崩れても問題ない
  - 事業計画書（★★★☆☆）: 数字が多少ずれても修正可能
  - **プログラム開発（★★★★★）**: 一文字でも間違えればエラー → だからこそ訓練素材として最適
