## 核心メッセージ

### このコラムで伝えるべき本質（1文）
**「AIは万能ではなく、あなたの背景・状況・意図を正しく与えなければ的外れな出力になる（Garbage In, Garbage Out）。だからこそコンテクストコントロール能力が必要で、それを鍛える最高の訓練題材がプログラミング（本当に動くものを作る）なのだ」**

### 読後に読者に残すべき3つのポイント
1. **Garbage In, Garbage Out**: AIは世の中の知識は知っているが、あなたの背景は知らない。だから正しいコンテクストを与えることが最重要
2. **コンテクストコントロール能力**: 正しく与える、たくさん与える、いつでも引き出せる、提供量と品質を担保する—この4つが本当のゴール
3. **なぜプログラミングか**: 企画書で終わらない「本当に動くもの」の説得力は全然違う。そして「一文字でも間違えればエラー」という厳しさが完璧な訓練になる

### 導入部（400-500字）
**狙い**: 「AIに指示したけど思った通りにならない」という読者の経験から始め、Garbage In, Garbage Outの本質を示す

**含めるべき要素**:

- 「AIって使えないじゃん」と思ったことはないか？

- **本当のゴールは「コンテクストコントロール能力」**

**書き出し例**:
ChatGPTに「企画書を作って」と頼んだけど、なんか違う。思ってたのと全然違う。「AIって使えないじゃん」—そう思ったことはありませんか？

でも、ちょっと待ってください。**それはAIが悪いのではなく、あなたの「与え方」が足りないだけです。**

AIは世の中の知識は膨大に知っています。でも、**あなたの背景は知りません**。なぜそのアイデアなのか、普段どんな生活をしているか、この会社でどう実現しなきゃいけないか—そういう背景を与えないと、的外れな出力になるのは当然です。

これを「**Garbage In, Garbage Out（ゴミを入れたらゴミが出る）**」と言います。

### 本論部（1,200-1,400字）

**展開の流れ**:

1. **コンテクストコントロール能力とは何か**（300-400字）:
   - ゴールはプログラミング習得ではない
   - ゴールは**コンテクストコントロール能力**：
     - **正しく与える**: 的確な情報を提供する
     - **たくさん与える**: 十分な量の背景情報を提供する
     - **いつでも引き出せるようにしておく**: ドキュメント化し、保管・提供の仕組みを作る
     - **提供量と提供品質を担保する**: 継続的に品質を管理する
   - これができれば、AIから期待通りの出力が得られる
   - これができなければ、どんなに高性能なAIでもGarbage In, Garbage Out

2. **なぜプログラミングを「題材」に選んだのか**（500-600字）:

   - **理由1: 企画書で終わらない、本当に動くものの説得力**
     - 企画書だけで終わる vs **本当に動くプロトタイプ**
     - 非エンジニアだからこそ、動くものを見せる説得力が全然違う
     - vibeCodingは「プログラミングを学ぶ」のではなく「動くものを作る手段」

   - **理由2: コンテクストコントロールを鍛える最高の訓練**
     - プログラムは「**一文字でも間違えればエラー**」
     - 曖昧さが一切許されない
     - だからこそ、AIに「正しく、たくさん、引き出せる形で」与える訓練に最適
     - Garbage In, Garbage Outを最も痛感できる

   - **理由3: ビジネス文書は「余裕」になる**
     - 難易度の階層：
       ```
       | タスク | 難易度 |
       |--------|--------|
       | プログラム開発（動くもの） | ★★★★★ |
       | 事業計画書 | ★★★☆☆ |
       | プレゼン資料 | ★★☆☆☆ |
       | メール文章 | ★☆☆☆☆ |
       ```
     - **最も難しいプログラミングでコンテクストコントロールを身につければ、他は比較的容易**

3. **非エンジニアが直面する「壁」**（300-400字）:
   - アイデアが生まれる → でも自分では作れない → 誰かに頼む → お金が必要
   - **「他力本願の苦しみ」**：
     - 毎日アイデアが生まれるのに形にできない（10年間の経験）
     - エンジニアに依存、外注に依存、資金調達に依存
     - **企画書だけでは伝わらない。本当に動くものが見せられたら…**
     - アイデアが形になるまでのステップが遠すぎる

4. **vibeCodingがもたらす革命**（200-300字）:
   - アイデアが生まれる → Cursorを開く → AIに（コンテクストを正しく与えて）指示 → 数時間〜数日で動くものが形になる
   - **他力本願から自律創造へのパラダイムシフト**
   - 企画書で終わらない、本当に動くプロトタイプの説得力

**必須で含める具体例・メタファー**:

  - プログラムの完璧性要求が、Garbage In, Garbage Outを防ぐ訓練になる

### 実践パート（200-300字）
