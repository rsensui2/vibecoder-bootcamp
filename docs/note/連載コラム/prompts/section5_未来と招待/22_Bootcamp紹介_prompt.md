### 本論部1（600-700字）: Bootcampの3つの特徴

**狙い**: Bootcampの具体的な価値を示し、読者が「自分も参加したい」と思えるようにする

#### 特徴1: 企業向けと同じカリキュラム（180万円の価値）

**企業向けプログラムの実績**:
- 武蔵野大学アントレプレナーシップ学部で採用
- 大手企業の新規事業部門で導入
- 1人あたり180万円の投資価値

**カリキュラム内容**:
- **Week 1-2**: Context Controlの4要素、要件定義の5ステップ
- **Week 3-4**: Cursor実践、データベース設計、Supabase実装
- **Week 5-6**: 認証システム（Clerk）、デプロイ（Vercel）、本番環境構築
- **Week 7**: 最終プロジェクト発表、フィードバック

**180万円の価値とは**:
- 企業にとってのROI: 新規事業の立ち上げ速度が10倍、開発コストが1/10
- 個人にとってのキャリア資産: アイデアを形にする力、AI時代のリテラシー

#### 特徴2: 非エンジニア特化の設計

**非エンジニアの3つの不安**:
1. 「プログラミング経験ゼロで大丈夫？」
2. 「技術用語がわからない」
3. 「挫折しそう」

**Bootcampの解決策**:
1. **プログラミング不要**: コードを書かない。AIと対話する能力を磨く
2. **比喩中心の説明**: レストラン、図書館、ホテルの鍵...日常的な例で理解
3. **段階的な学習**: 小さな成功体験を積み重ねる設計

**実例: 「いいね」機能を作る**
- Day 1: 要件定義（「いいね」は何のため？誰のため？）
- Day 3: UI設計（v0で画面を生成）
- Day 7: DB設計（Supabaseでlikesテーブルを作成）
- Day 14: 認証連携（Clerkでユーザー管理）
- Day 21: デプロイ（Vercelで世界に公開）

**この過程で、一度もコードを書かない**

#### 特徴3: 挫折させないサポート体制

**3つのサポート**:
1. **毎週のライブセッション**: 泉水亮介による直接指導、Q&A、デモ
2. **専用コミュニティ**: Slackで24時間質問可能、仲間と学び合う
3. **個別メンタリング**: 1on1で進捗確認、挫折ポイントを事前に解消

**挫折させない設計思想**:
- 「わからない」をそのままにしない
- 「できた」という成功体験を毎週積む
- 「仲間がいる」という心理的安全性

## 生成時の注意事項

### Context Controlとの接続
- Bootcampは「Context Controlの4要素」を7週間で体得するプログラム
- 「正しく与える」→ 要件定義の5ステップ
- 「たくさん与える」→ 知識の構造化、プロンプト設計
- 「いつでも引き出せるようにしておく」→ GitHub、ドキュメント管理
- 「提供量と提供品質を担保する」→ Vibes Output Loop、品質チェック

### Garbage In, Garbage Out
- Bootcampでは、「AIに何を伝えるか」を学ぶ
- 曖昧な指示 → 曖昧な出力
- 具体的な要件定義 → 高品質な実装

### 「本当に動くもの」を作る
- Bootcampの成果物は、すべて「本番環境で動くWebアプリ」
- 企画書ではなく、実際にデプロイされたアプリ
- ポートフォリオとして使える
