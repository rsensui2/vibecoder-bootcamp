#### ステップ1: ビジネス要件定義（誰のために、なぜ）
**人間が考える部分**:
- 誰のために（ターゲットユーザー）
- なぜ（どんな課題を解決するか）
- どんな価値を提供するか

**StudyCircle SNS例**:
- 誰: 大学生、社会人学習者
- なぜ: 学習を継続するのが難しい、孤独、モチベーション維持が困難
- 価値: 仲間と一緒に学習することで、モチベーション維持と継続をサポート

**AIへの質問例**:
「大学生や社会人学習者が、学習を継続しやすくするためのSNSアプリを作りたい。どんな機能があれば、モチベーション維持に繋がるでしょうか？」

#### ステップ2: 機能要件定義（何を作るか）
**AIとの対話**: ステップ1のビジネス要件をもとに、具体的な機能を洗い出す

**StudyCircle SNS例**:
- 学習投稿機能（今日学んだことをシェア）
- コメント・いいね機能（仲間からのフィードバック）
- 学習グループ機能（同じ目標を持つ仲間と繋がる）
- 学習記録機能（継続日数、学習時間の可視化）

**AIへの質問例**:
「これらの機能で、モチベーション維持に効果的なものはどれですか？優先順位をつけてください」

#### ステップ3: UI/UX設計（どう見せるか）
**AIとの対話**: 機能要件をもとに、ユーザーが使いやすいUI/UXを設計

**StudyCircle SNS例**:
- トップページ: タイムライン形式で仲間の学習投稿を表示
- 投稿詳細: コメント、いいね、学習記録を表示
- マイページ: 自分の学習記録、継続日数、達成バッジ

**AIへの質問例**:
「学習SNSのトップページのワイヤーフレームを作ってください。タイムライン形式で、投稿、コメント、いいねが見えるように」

**ここでv0を活用**: AIが生成したワイヤーフレームをv0で実際のUIに変換

#### ステップ4: 技術スタック選定（何で作るか）
**AIとの対話**: 機能要件、UI/UX要件をもとに、最適な技術スタックを選定

**StudyCircle SNS例**:
- フロントエンド: Next.js 14 (App Router)
- バックエンド: Supabase（PostgreSQL + リアルタイム同期）
- 認証: Clerk（Google認証、GitHub認証）
- デプロイ: Vercel

**AIへの質問例**:
「学習SNSを作るのに最適な技術スタックを提案してください。リアルタイム性、スケーラビリティ、開発速度を重視します」

#### ステップ5: 詳細設計（どう実装するか）
**AIとの対話**: 技術スタックが決まったら、データベース設計、API設計、コンポーネント設計を詳細化

**StudyCircle SNS例**:
- データベース設計: users, posts, comments, likes, groupsテーブル
- API設計: Server Actions for CRUD操作
- コンポーネント設計: PostCard, CommentList, UserProfile等

**AIへの質問例**:
「学習SNSのデータベース設計を提案してください。ER図も含めて」

### 本論部2（400-500字）: 各ステップでのContext Controlの実践

**狙い**: 5ステップが単なるプロセスではなく、Context Controlの実践であることを示す

#### Context Controlの4要素との対応
1. **正しく与える**: 各ステップで、AIに明確な質問をする（曖昧な指示ではなく、具体的な要件）
2. **たくさん与える**: ステップ1からステップ5まで、徐々にコンテクストを積み重ねる
3. **いつでも引き出せるようにしておく**: 各ステップの成果物をドキュメント化（要件定義書、ER図、ワイヤーフレーム等）
4. **提供量と提供品質を担保する**: AIのフィードバックをもとに、要件を洗練させる

#### Garbage In, Garbage Outの回避
- ステップ1が曖昧だと、ステップ2以降も曖昧になる
- だからこそ、ステップ1（ビジネス要件）を丁寧に明確化することが最重要
- AIは「なぜそれが必要か」を理解して初めて、最適なシステム要件を提案できる

### 実践パート（300-400字）: 今日からできる「5ステップ実践」

**狙い**: 読者がすぐに試せる実践的なアクションを提示

**実践ハック: あなたのアイデアで5ステップを試す**
1. **ステップ1**: 5W1Hでビジネス要件を整理（前回の実践ハック）
2. **ステップ2**: ChatGPTに「こういう課題を解決したい。どんな機能が必要？」と質問
3. **ステップ3**: v0.devで「こういう機能のトップページUI」を生成
4. **ステップ4**: ChatGPTに「この機能を実装するのに最適な技術スタックは？」と質問
5. **ステップ5**: ChatGPTに「データベース設計を提案して」と質問

**重要**: 各ステップでAIの回答を保存し、次のステップのコンテクストとして活用する

### 結論部（200-300字）: Context Controlとの繋がり、次回への誘導

**狙い**: このコラムの内容をContext Controlの本質に結びつけ、LLMの仕組み理解へ誘導

**含めるべき要素**:
