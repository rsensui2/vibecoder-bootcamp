# #1「プログラミングを学ぶな」という挑発

## AIに裏切られた経験、ありませんか？

ChatGPTに「企画書を作って」と頼んだけど、なんか違う。思ってたのと全然違う。「AIって使えないじゃん」—そう思ったことはありませんか？

でも、ちょっと待ってください。**それはAIが悪いのではなく、あなたの「与え方」が足りないだけです。**

AIは世の中の知識は膨大に知っています。Wikipediaも、過去の論文も、ビジネス事例も、何でも知っている。でも、**あなたの背景は知りません**。なぜそのアイデアを思いついたのか、普段どんな生活をしているか、あなたの会社でどう実現しなきゃいけないか—そういう「あなた固有の文脈」を与えないと、的外れな出力になるのは当然です。

これを「**Garbage In, Garbage Out（ゴミを入れたらゴミが出る）**」と言います。

AIは万能ではありません。あなたが曖昧な指示を出せば、AIも曖昧な答えしか返せない。あなたが背景を説明せずに「いい感じにして」と言えば、AIは「いい感じって何？」と困惑します（表面上は困惑を見せませんが）。

だから、本当のゴールは「**コンテクストコントロール能力**」を身につけることなのです。

## コンテクストコントロール能力とは何か

ゴールはプログラミング言語を習得することではありません。ゴールは**コンテクストコントロール能力**です。

これは4つの要素から成り立っています：

1. **正しく与える**: 的確な情報を提供する
2. **たくさん与える**: 十分な量の背景情報を提供する
3. **いつでも引き出せるようにしておく**: ドキュメント化し、保管・提供の仕組みを作る
4. **提供量と提供品質を担保する**: 継続的に品質を管理する

この4つができれば、AIから期待通りの出力が得られます。逆に、この4つができなければ、どんなに高性能なAI（GPT-4だろうが、Claude Sonnetだろうが）を使っても、Garbage In, Garbage Outです。

「でも、それってプログラミングと何の関係があるの？」と思いましたよね。

実は、**プログラミングこそが、このコンテクストコントロール能力を鍛える最高の訓練題材**なのです。しかも、それだけではない。もっと強力な理由があります。

## なぜプログラミングを題材に選んだのか

### 理由1: 企画書で終わらない、本当に動くものの説得力

あなたは今まで、何枚の企画書を書いてきましたか？

その企画書は、どれだけ実現しましたか？

企画書は素晴らしいツールです。でも、**企画書だけで終わる**ことが、あまりにも多い。

なぜか？

企画書は「こういうものを作りたい」という**意図**は伝えられますが、**実現可能性**は証明できないからです。読む人は想像するしかない。「本当にこれ、できるの？」という疑問が常につきまとう。

でも、もしあなたが**本当に動くプロトタイプ**を見せられたら？

「ほら、これです。実際に動きます。ログインもできます。データも保存されます。スマホでも見られます」

説得力が、**全然違います**。

非エンジニアだからこそ、動くものを見せる説得力が圧倒的なのです。「この人、エンジニアじゃないのに、ここまで作ったの？」という驚きが、信頼に変わります。

vibeCodingは「プログラミングを学ぶ」のではありません。**企画書で終わらない、本当に動くものを作る手段**なのです。

### 理由2: コンテクストコントロールを鍛える最高の訓練

プログラムには、ある特徴があります。

**一文字でも間違えればエラー**

これです。

ビジネス文書なら、多少の誤字脱字があっても意味は通じます。「だいたいこういうこと言いたいんだろうな」と読み手が補完してくれる。

でも、プログラムは違います。

```javascript
console.log("Hello World");  // これは動く
console.log("Hello World");  // これもエラー（全角スペース）
```

たった1文字、全角スペースが混じっただけで、エラーです。動きません。容赦なく止まります。

この**曖昧さが一切許されない厳しさ**が、コンテクストコントロールを鍛える完璧な訓練になるのです。

AIに「ログイン機能を作って」と曖昧に指示しても、AIは困ります（Garbage In）。

「誰がログインするのか（ユーザー種別）」「何でログインするのか（メールアドレス？電話番号？）」「ログイン後にどこに遷移するのか（ダッシュボード？マイページ？）」「パスワードを忘れた場合はどうするのか」—こういう背景を**正しく、たくさん与える**必要があります。

一文字でもエラーが出る厳しい世界だからこそ、Garbage In, Garbage Outを最も痛感できる。だからこそ、最高の訓練になるのです。

### 理由3: ビジネス文書は「余裕」になる

実は、タスクには難易度の階層があります。

| タスク | 難易度 |
|--------|--------|
| プログラム開発（動くもの） | ★★★★★ |
| 事業計画書 | ★★★☆☆ |
| プレゼン資料 | ★★☆☆☆ |
| メール文章 | ★☆☆☆☆ |

プログラム開発は、**最も難易度が高い**タスクです。

なぜなら、「一文字でも間違えればエラー」「全ての要素が正しく繋がらないと動かない」「曖昧さが一切許されない」からです。

でも、逆に言えば、**最も難しいプログラミングでコンテクストコントロールを身につければ、他のタスクは比較的容易**になるのです。

事業計画書？ AIに背景を正しく与えれば、すぐに作れます。
プレゼン資料？ 余裕です。
メール文章？ 秒で終わります。

プログラミングという「最高難度の訓練」を経験すれば、ビジネス文書は「余裕」になる。これが、vibeCoding Bootcampが企業研修として1人あたり180万円の価値がある理由の一つです。

### 実践1: ChatGPTに曖昧な指示 vs 背景を含む指示

まず、ChatGPTを開いて、こう指示してみてください：

**「企画書を作って」**

どうなりましたか？ おそらく、汎用的な企画書のテンプレートが返ってきたはずです。使えないことはないけど、「これじゃない」感が強いはずです。

次に、こう指示してみてください：

**「私は中小企業の経営者です。従業員50人の製造業で、現在、生産管理が紙ベースで非効率です。DX化を進めたいのですが、IT部門がないため、外部のシステム会社に依頼する必要があります。取締役会で承認を得るための企画書を作ってください。読者は経営陣（平均年齢60歳、IT知識は初級レベル）です。A4で3ページ以内、専門用語は避けて、ROIを明確に示す内容にしてください」**

どうでしょう？ 期待に近い企画書が返ってきませんでしたか？

これが、**コンテクストコントロールの第一歩**です。

### 実践2: 「何を作りたいか」を背景も含めて3行で書き出す

次に、あなたが「作りたいアプリ」を、背景も含めて3行で書き出してみてください。

- **なぜそれを作りたいのか**
- **どういう状況で使うのか**
- **どんな人が使うのか**

これが「正しく、たくさん与える」の訓練です。

この3行を書ければ、AIに指示を出す準備ができています。
