# 4-4 デモデイと最終課題

## 概要
Vibe Coder Bootcampの集大成として、これまで学んだ技術とスキルを活かしてオリジナルWebアプリケーションを開発し、デモデイで発表していただきます。この章では、デモデイの目的、最終課題の内容、評価基準、そして技術の応用方法について詳しく説明します。

## 学習目標
- デモデイの目的とゴールを理解する
- 最終課題の内容と進め方を把握する
- AIを活用した効率的な開発プロセスを実践する
- 学んだ技術をコーディング以外の業務にも応用する方法を考える

# 1. デモデイの目的：あなたの「できた！」を共有する場
デモデイは、単なる成果発表会ではありません。

- **成功体験の共有**: あなたが「アイデアを自らの手で実現した」というVibe Coderとしての成功体験を分かち合う場です。
- **学びの深化**: 他の受講生の作品やアプローチから学び、新たなインスピレーションを得る場です。
- **未来への一歩**: この経験を自信に、次のプロダクト開発やキャリアへの一歩を踏み出すためのキックオフです。
- **技術の応用**: コーディング以外の業務にもAIを活用する方法を共有し、組織全体の生産性向上に貢献します。

# 2. 最終課題：オリジナルWebアプリケーションの開発

## 課題の目的
- **知識の統合**: フロントエンド、バックエンド、データベース、認証、デプロイという一連の技術を統合し、一つのプロダクトとして完成させます。
- **実践力の証明**: AI駆動開発のプロセスを自律的に実践し、「アイデアを形にする力」を証明します。
- **ポートフォリオの作成**: デモデイで発表する作品は、あなたのスキルを証明する貴重なポートフォリオとなります。

## 課題内容
**テーマは自由です。** あなたが「本当に創りたい」と思うもの、「誰かの課題を解決したい」と考えるものを形にしてください。SNSアプリケーションの延長線上のアイデアをおすすめします。

### 必須要件
以下の技術要素をすべて含んだWebアプリケーションを開発してください。

1. **フロントエンド**: ユーザーが操作するUIがあること。（Next.js + TypeScriptを推奨）
2. **バックエンド**: サーバーサイドでのデータ処理があること。（Next.js API Routesを推奨）
3. **データベース**: データの永続化を行っていること。（Supabaseを推奨）
4. **認証**: ユーザー登録・ログイン機能があること。（Clerkを推奨）

### ゴール設定
あなたのスキルレベルや確保できる時間に応じて、以下のいずれかをゴールとして設定してください。

- **理想ゴール**: 開発したアプリケーションをVercelに**デプロイ**し、公開URLを提出する。実際に触ってもらえる。
- **最低限ゴール**: **ローカル環境でフロントエンドが動作する**状態にし、そのデモができるように準備する。

# 3. 開発プロセス
これまでの講義で学んだAI駆動開発のフローを実践しましょう。

## 1. 要件定義
AIと対話し、創りたいものの要件をMarkdownで詳細に定義します。


## 2. 設計
AIに要件を渡し、DB設計、API設計、UI設計などを行います。



## 3. 実装
設計書を基に、AIに指示を出して段階的に機能を実装します。


## 4. テスト・デバッグ
実装した機能が正しく動作するか確認します。


## 5. デプロイ（理想ゴール）
Vercelにデプロイして、アプリケーションを公開します。

# 4. 発表形式：Vibe Coderスタイルを貫く
**PowerPoint, Keynote等のスライド作成ソフトは使用禁止**です。Vibe Coderらしく、プレゼンテーションもAIを活用してコードで作成しましょう。
Markdownをベースに内容を構築し、Gensparkなどのサービスを使うか、そのままHTMLでプレゼンスライドにしてしまいましょう。

- **発表時間**: 1人15分（発表10分、質疑応答3分を想定）
- **使用ツール**: **Cursor**
- **発表方法**:
    1. **ライブデモ**: 実際に開発したアプリケーションを動かしながらデモを行う。
    2. **プレゼン資料**: AIと共に作成したプレゼンテーションを表示しながらプレゼンする。


# 5. 評価基準：技術力よりも「魅力」
デモデイでは、コードの複雑さや技術的な難易度は評価の主軸ではありません。以下のユーザー視点・ビジネス視点を重視します。

- **アイデアの独創性**: 「面白い！」「使ってみたい！」と思わせるか？
- **UI/UXの魅力**: 直感的で、触っていて楽しいか？
- **課題解決**: 誰かの「困った」を解決しているか？
- **熱意とストーリー**: なぜこれを作ろうと思ったのか、その情熱が伝わるか？
- **技術の応用**: 学んだ技術やプロセスを他の業務やプロジェクトにどう活かせるか？
- **AI活用の工夫**: AIをどのように効果的に活用したか？

技術的な完成度よりも、あなたの「想い」と「学びの応用」が詰まった発表を楽しみにしています。

# 6. 技術の応用：Vibe Codingの先にあるもの
Vibe Coder Bootcampで学んだスキルは、プログラミングの枠を超えて様々な業務に応用できます。以下のような活用例を参考に、あなたなりの応用方法を考えてみてください。

## 1. ドキュメンテーションの効率化
- **AIを活用した議事録作成**: 会議の録音やメモから、AIを使って構造化された議事録を自動生成
- **要件定義書の作成**: 曖昧な要件をAIと対話しながら明確化し、仕様書として整理
- **マニュアル作成**: コードやプロセス説明をAIに依頼して、分かりやすいドキュメントを作成

## 2. 業務プロセスの改善
- **定型業務の自動化**: メールの自動返信やデータ収集など、反復作業をスクリプト化
- **データ分析の効率化**: 大量のデータからAIに傾向分析やレポート作成を依頼
- **プロジェクト管理**: タスクの進捗管理やリスク予測をAIにサポートしてもらう

## 3. アイデア創出・意思決定支援
- **ブレインストーミング**: AIを対話的なアイデア出しのパートナーとして活用
- **意思決定の補助**: 複数の選択肢をAIに評価・比較してもらい、意思決定をサポート
- **市場調査**: 新規事業やプロダクトのアイデアをAIに評価・改善提案してもらう

## 4. コミュニケーションの質向上
- **メール・チャットの文面作成**: ビジネスメールや報告書の下書きをAIに作成依頼
- **プレゼン資料の作成**: 発表内容をAIに伝えて、効果的なスライド構成や表現を提案してもらう
- **フィードバックの分析**: 顧客やチームからのフィードバックをAIに分析してもらい、改善点を抽出

## 5. 学びの継続とスキルアップ
- **技術記事の要約**: 最新技術の記事をAIに要約・解説してもらう
- **コードレビューの補助**: 書いたコードをAIにレビューしてもらい、改善点を提案してもらう
- **学習計画の作成**: スキルアップのための学習ロードマップをAIと一緒に作成

# 7. デモデイ発表で共有してほしいこと
以下のような観点で、あなたの学びや気づきを共有してください：

1. **プロダクトの紹介**
   - どんなものを作ったか
   - なぜそれを作ろうと思ったか
   - どのようにして作ったか（特にAIをどう活用したか）

2. **学びの応用**
   - この経験を今後の業務にどう活かせるか
   - 他のプロジェクトや業務で試してみたい活用方法
   - チームや組織に広めたい使い方

3. **気づきや発見**
   - 実際に使ってみて分かったこと
   - 想像と違った点や意外な発見
   - もっと早く知りたかったこと
   - 今までの仕事の仕方とこれからの仕事、何が違うか？
# 8. 提出物と締切

## 提出物
1. **GitHubリポジトリのURL**: 開発の履歴が残るようにしてください。
2. **（可能であれば）デプロイしたアプリケーションのURL**
3. **デモデイ用プレゼン資料**: Markdown or HTML形式で作成したもの。

## 提出期限
**デモデイ前日**までに提出してください。

# 9. 困ったときのヒント

## 開発の壁にぶつかったら
- **Gitで過去に戻る**: 「この機能を追加したら動かなくなった…」という時は、Gitの力で動いていた時点に戻ることができます。慌てずに、安全なセーブポイントからやり直しましょう。

- **AIにエラーを解決させる**: エラーメッセージは最高のヒントです。エラー文をそのままコピーしてAIに「このエラーの原因と解決策を教えて」と聞きましょう。

## Slackでの質問
- 講師に気軽に質問できることを再度案内します。

# 10. 最後に
Vibe Coder Bootcampで学んだことは、単なるプログラミングスキルだけではありません。AIを効果的に活用し、アイデアを形にする力、問題を解決する力、そして新しい技術を学び続ける姿勢こそが、これからの時代に最も求められるスキルです。

この最終課題を通じて、ぜひ「自分にもできた！」という自信を手に入れてください。そして、その経験と学びを、今後の業務やキャリアに活かしていってほしいと思います。

皆さんの素晴らしい作品を楽しみにしています！
