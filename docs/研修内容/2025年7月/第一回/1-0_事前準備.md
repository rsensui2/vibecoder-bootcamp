## 3. macOS セットアップ（完全初心者向け・Apple Silicon対応）

### 📖 用語解説
- **Homebrew（ホームブリュー）**: macOS用のパッケージマネージャ。アプリやツールを簡単にインストール・管理できる
- **Apple Silicon**: M1/M2/M3チップのこと。Intelチップとは異なる設定が必要
- **ターミナル**: コマンドを入力してコンピュータを操作するアプリ（黒い画面）
- **PATH（パス）**: コンピュータがコマンドを探す場所のリスト

### ステップ1: Xcode Command Line Tools のインストール
開発に必要な基本ツール群をインストールします。

```bash
# ターミナルを開いて実行（Applications > Utilities > Terminal）
xcode-select --install
```

> 💡 **ヒント**: ポップアップが表示されたら「インストール」をクリック。約10分かかります。

### ステップ2: Homebrew のインストール（2025年9月最新手順）

#### 2-1. インストールコマンド実行
```bash
# Homebrewの公式インストーラを実行
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
```

#### 2-2. PATHの設定（Apple Silicon必須）
**重要**: Apple Silicon Mac（M1/M2/M3）では追加設定が必要です。

```bash
# zsh（macOSデフォルトシェル）の設定
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile
echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zshrc

# 設定を即座に反映
eval "$(/opt/homebrew/bin/brew shellenv)"
```

#### 2-3. インストール確認
```bash
# Homebrewが正しくインストールされたか確認
brew --version
which brew  # /opt/homebrew/bin/brew と表示されればOK（Apple Silicon）
brew doctor # 「Your system is ready to brew.」と表示されれば成功
```

### ステップ3: 開発ツールの一括インストール

```bash
# 必要なツールをまとめてインストール
brew install node@22 git pnpm vercel supabase/tap/supabase

# Node.js 22をデフォルトに設定
brew link node@22 --force --overwrite
```

> ⚠️ **注意**: `supabase/tap/supabase`の部分は、Supabaseの公式tapリポジトリを指定しています。

### ステップ4: Cursor のインストールと設定

1. **ダウンロード**
   - [https://www.cursor.com](https://www.cursor.com) にアクセス
   - 「Download for Mac」をクリック
   - Apple Silicon用（M1/M2/M3）かIntel用か自動判定されます

2. **インストール**
   - ダウンロードした`.dmg`ファイルをダブルクリック
   - Cursorアイコンを`Applications`フォルダにドラッグ&ドロップ

3. **初回起動と設定**
   - Applicationsフォルダから「Cursor」を起動
   - 「Sign in with GitHub」または「Sign in with Google」でログイン
   - Settings > Billing から「Pro」プランにアップグレード（$20/月）

### ステップ5: 環境確認チェックリスト


## 4. Windows セットアップ（2つの方法から選択）

### 🔴 最重要：PowerShell実行ポリシーの設定（Windows必須）

**Windowsユーザーは必ず最初にこの設定を行ってください。**これを行わないと、npm、pnpm、yarn等のコマンドで「UnauthorizedAccess」エラーが発生します。

#### PowerShell実行ポリシーとは？
WindowsのPowerShellは、セキュリティのためデフォルトでスクリプトの実行を禁止しています。npmやその他のNode.js関連ツールは内部でPowerShellスクリプト（.ps1ファイル）を使用するため、この制限を緩和する必要があります。

#### 設定手順（全Windowsユーザー必須）

1. **PowerShellを管理者権限で起動**
   - スタートボタンを右クリック
   - 「Windows Terminal (Admin)」または「Windows PowerShell (管理者)」を選択

2. **現在の実行ポリシーを確認**
   ```powershell
   Get-ExecutionPolicy
   ```
   おそらく「Restricted」と表示されます。

3. **実行ポリシーを変更**
   ```powershell
   # 現在のユーザーに対してRemoteSignedポリシーを設定
   Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
   ```

4. **確認プロンプトへの応答**
   ```
   実行ポリシーを変更しますか？
   [Y] はい(Y)  [A] すべて続行(A)  [N] いいえ(N)  [L] すべて無視(L)  [S] 中断(S)  [?] ヘルプ (既定値は "N"):
   ```
   **「A」を入力してEnterキーを押してください。**

5. **設定が適用されたか確認**
   ```powershell
   Get-ExecutionPolicy -Scope CurrentUser
   # 「RemoteSigned」と表示されればOK
   ```

#### なぜRemoteSignedが安全なのか？
- **RemoteSigned**：ローカルで作成されたスクリプトは実行可能、インターネットからダウンロードされたスクリプトには信頼できる発行元の署名が必要
- **CurrentUserスコープ**：現在のユーザーアカウントにのみ適用され、システム全体には影響しない
- **開発に最適**：セキュリティを保ちつつ、開発作業に必要な柔軟性を提供

#### 会社PCで権限がない場合の対処法

会社のセキュリティポリシーで実行ポリシーの変更が禁止されている場合：

**対処法1：npxを使用（推奨）**
```powershell
# グローバルインストールせずに直接実行
npx vercel --version
npx supabase --version
npx create-next-app@latest my-app
```

**対処法2：コマンドプロンプト（cmd）を使用**
```cmd
# cmdはPowerShellの実行ポリシーの影響を受けない
# スタート > 「cmd」と入力 > Enter
npm install -g vercel
npm install -g supabase
```

**対処法3：IT部門に相談**
開発業務で必要な旨を説明し、実行ポリシーの変更許可を申請

### 📖 用語解説
- **PowerShell**: Windows用の高機能コマンドラインツール。Windows 11標準は5.1
- **PowerShell 7**: クロスプラットフォーム対応の最新版PowerShell（任意でアップグレード可能）
- **winget**: Windows公式のパッケージマネージャ（Windows Package Manager）
- **WSL2**: Windows上でLinuxを動かす仕組み（Windows Subsystem for Linux 2）
- **管理者権限**: システムの重要な部分を変更する権限

### 方法A: Windows環境での開発（簡単・初心者向け）

#### ステップ1: PowerShellを管理者権限で起動
1. スタートボタンを右クリック
2. 「Windows Terminal (Admin)」または「Windows PowerShell (管理者)」を選択

#### ステップ2: wingetの確認と更新
```powershell
# wingetが利用可能か確認
winget --version

# 利用できない場合はMicrosoft Storeから「App Installer」をインストール
# または以下のコマンドで更新

```

#### ステップ3: 開発ツールのインストール
```powershell
# Node.js LTS（推奨）
winget install --id OpenJS.NodeJS.LTS -e --accept-source-agreements --accept-package-agreements

# pnpm（高速パッケージマネージャ）
winget install --id pnpm.pnpm -e --accept-source-agreements --accept-package-agreements

# Git（バージョン管理）
winget install --id Git.Git -e --accept-source-agreements --accept-package-agreements

# Windows Terminal（便利なターミナル）
winget install --id Microsoft.WindowsTerminal -e --accept-source-agreements --accept-package-agreements

# （任意）PowerShell 7 - より高速で安定した最新版
winget install --id Microsoft.PowerShell -e --accept-source-agreements --accept-package-agreements
```

#### ステップ4: 追加ツールのインストール
```powershell
# ターミナルを再起動してから実行

# Vercel CLI
npm install -g vercel

# Supabase CLI  
npm install -g supabase
```

> ⚠️ **エラーが出た場合**: 上記の「PowerShell実行ポリシーの設定」を確認してください。

#### （任意）PowerShell 7への切り替え
PowerShell 7は、Windows標準の5.1より高速で、Cursor IDEとの互換性も向上します。

```powershell
# インストール後、Cursorの設定で変更
# Settings > Terminal > Default Profile > PowerShell 7
```

1. [https://supabase.com/dashboard](https://supabase.com/dashboard) にログイン
2. プロジェクト選択 > Settings > API
3. 「anon public」キーをコピー（後で使用）
4. 「Project URL」をコピー（後で使用）

## 8. トラブルシューティング（詳細版）

### 🔧 Windows PowerShell のトラブル（最重要）

| 症状 | 原因 | 解決方法 |
| ---- | ---- | -------- |
| `UnauthorizedAccess`エラー | 実行ポリシーがRestricted | 管理者PowerShellで：<br>`Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser`<br>「A」を入力してEnter |
| `npm: command not found` | PATHが設定されていない | 1. Node.jsを再インストール<br>2. PCを再起動<br>3. `$env:Path`でNode.jsのパスを確認 |
| wingetが使えない | App Installerが古い | Microsoft Storeから「App Installer」を更新 |
| 会社PCで実行ポリシー変更不可 | グループポリシー制限 | 1. npxを使用（推奨）<br>2. cmdを使用<br>3. IT部門に相談 |
| Cursorでターミナルエラー | PowerShell 5.1の互換性問題 | PowerShell 7をインストールし、デフォルトに設定 |
| スクリプトエラーが頻発 | 文字コードの問題 | ターミナル設定で「UTF-8」を使用 |

### 🔧 macOS のトラブル

| 症状 | 原因 | 解決方法 |
| ---- | ---- | -------- |
| `command not found: brew` | PATHが設定されていない | 以下を実行：<br>`echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile`<br>`eval "$(/opt/homebrew/bin/brew shellenv)"` |
| `node: command not found` | Node.jsのリンクが切れている | `brew link --overwrite node@22` |
| Homebrewが`/usr/local`にインストールされた | Rosettaモードで実行した | ターミナルの情報を見て「Rosettaを使用して開く」のチェックを外す |
| `brew doctor`でWarning多数 | 古い設定が残っている | `brew cleanup`を実行、それでも解決しない場合は個別に対処 |
| Apple Siliconなのにx86_64と表示 | Rosettaモードで動作中 | `uname -m`で確認。`arch -arm64 zsh`で native shellを起動 |

### 🔧 WSL2 のトラブル

| 症状 | 原因 | 解決方法 |
| ---- | ---- | -------- |
| `wsl --install`が失敗 | 仮想化が無効 | BIOS/UEFIで仮想化（Intel VT-x/AMD-V）を有効化 |
| WSL内でネットが繋がらない | DNS設定の問題 | WSL内で：<br>`echo "nameserver 8.8.8.8" | sudo tee /etc/resolv.conf` |
| CursorでWSLに接続できない | Remote-WSL拡張機能なし | 拡張機能マーケットプレイスから「Remote - WSL」をインストール |
| ファイル操作が遅い | /mnt/c/を使用している | プロジェクトを`/home/ユーザー名/`に移動 |

### 🔧 Cursor のトラブル

| 症状 | 原因 | 解決方法 |
| ---- | ---- | -------- |
| ログインループ | キャッシュの問題 | 1. Help > Clear Cache<br>2. ブラウザで https://cursor.com/reset<br>3. Cursorを再起動 |
| AI補完が動作しない | APIキー未設定またはPro未加入 | Settings > Billing でProプラン確認 |
| 「Too many requests」エラー | 使用量制限に到達 | 1. Settings > Usage で使用量確認<br>2. 翌月まで待つかプランアップグレード |
| ターミナルが文字化けする | 文字コード設定 | Settings > Terminal > Encoding を「UTF-8」に設定 |
| PowerShellでエラー多発 | PowerShell 5.1の問題 | PowerShell 7をインストールしてデフォルトに設定 |

### 🔧 Next.js のトラブル

| 症状 | 原因 | 解決方法 |
| ---- | ---- | -------- |
| `pnpm: command not found` | pnpmが未インストール | `npm install -g pnpm`または`corepack enable` |
| ポート3000が使用中 | 他のアプリが使用 | `pnpm dev -p 3001`で別ポート使用 |
| Module not foundエラー | 依存関係が不足 | `pnpm install`を実行 |
| Turbopackエラー | 非対応の設定 | `next.config.js`で`experimental.turbo`を削除 |

### 🔧 Git/GitHub のトラブル

| 症状 | 原因 | 解決方法 |
| ---- | ---- | -------- |
| Permission deniedエラー | SSH鍵が未設定 | [GitHub SSH鍵設定ガイド](https://docs.github.com/ja/authentication/connecting-to-github-with-ssh)を参照 |
| git pushが拒否される | 認証トークンの期限切れ | Personal Access Tokenを再生成 |
| コミットできない | ユーザー情報未設定 | `git config --global user.name "名前"`<br>`git config --global user.email "email@example.com"` |

## 9. 専門用語集（初心者向け）

### 基本用語
- **CLI（Command Line Interface）**: 文字でコンピュータを操作するインターフェース。GUIの反対
- **シェル（Shell）**: コマンドを解釈して実行するプログラム（PowerShell、bash、zshなど）
- **ターミナル（Terminal）**: シェルを実行するためのアプリケーション
- **パッケージマネージャ**: ソフトウェアのインストール・更新・削除を管理するツール
- **ランタイム**: プログラムを実行するための環境
- **フレームワーク**: アプリ開発の土台となる仕組み
- **API（Application Programming Interface）**: プログラム同士が通信するための仕組み
- **APIキー**: APIを使うための認証情報（パスワードのようなもの）

### Windows特有の用語
- **PowerShell**: Windows用の高機能コマンドラインツール
- **実行ポリシー（Execution Policy）**: PowerShellがスクリプトを実行する際のセキュリティ設定
- **管理者権限**: システムの重要な部分を変更できる特別な権限
- **winget**: Windows公式のパッケージマネージャ
- **WSL（Windows Subsystem for Linux）**: Windows上でLinuxを動かす仕組み

### 開発ツール用語
- **IDE（Integrated Development Environment）**: 統合開発環境。コード作成を支援するソフト
- **AI補完**: AIがコードを予測して提案する機能
- **バージョン管理**: コードの変更履歴を記録・管理するシステム
- **リポジトリ**: プロジェクトのコードや履歴を保存する場所
- **デプロイ**: 作ったアプリをインターネット上に公開すること
- **npx**: npmパッケージを一時的に実行するためのツール（インストール不要）

### Web開発用語
- **フロントエンド**: ユーザーが見て操作する部分（画面）
- **バックエンド**: サーバー側の処理部分（データ処理など）
- **フルスタック**: フロントエンドとバックエンド両方を扱うこと
- **BaaS（Backend as a Service）**: バックエンドの機能をサービスとして提供するもの
- **CDN（Content Delivery Network）**: コンテンツを高速配信する仕組み

### Next.js関連用語
- **SSR（Server Side Rendering）**: サーバー側でHTMLを生成する方式
- **SSG（Static Site Generation）**: ビルド時にHTMLを生成する方式
- **App Router**: Next.js 13以降の新しいルーティングシステム
- **Turbopack**: Next.jsの高速バンドラー（コードをまとめるツール）

## 10. よくある質問（FAQ）

### Q: PowerShellでnpmコマンドが動作しません
**A**: 実行ポリシーの設定が必要です。「4. Windows セットアップ」の冒頭にある「PowerShell実行ポリシーの設定」を実施してください。

### Q: 会社PCでセキュリティエラーが出ます
**A**: 以下の順番で試してください：
1. npxを使用する（例：`npx vercel`）
2. コマンドプロンプト（cmd）を使用する
3. IT部門に開発用途での制限緩和を相談する

### Q: bashコマンドがWindowsで使えません
**A**: bashはLinux/macOS用のコマンドです。Windowsでは：
- PowerShellまたはcmdのコマンドを使用
- WSL2をインストールしてLinux環境を構築
- Git Bashをインストール（Gitに付属）

### Q: Cursorのターミナルでエラーが多発します
**A**: PowerShell 7へのアップグレードを推奨します：
```powershell
winget install --id Microsoft.PowerShell
```
その後、Cursorの設定でデフォルトシェルをPowerShell 7に変更。

### Q: MacとWindowsでコマンドが違うのはなぜ？
**A**: OSによってデフォルトのシェルが異なるためです：
- macOS/Linux: bash/zsh（UNIX系）
- Windows: PowerShell/cmd（Windows系）
基本的な開発コマンド（npm、pnpm、git等）は共通です。
