# VibeCoder育成プログラム - 大学版7回構成カリキュラム

## 📋 全体概要

### プログラム基本情報
- **対象**: 大学生（プログラミング未経験者〜初心者）
- **総時間**: 7回 × 90分 = 10.5時間
- **開発手法**: AI駆動開発（自動プロンプトシステム活用）
- **最終成果物**: オリジナルWebアプリケーション + プレゼンテーション

### 企業版との主な違い
| 項目 | 企業版 | 大学版 |
|------|--------|--------|
| セッション数 | 4回 × 150分 | 7回 × 90分 |
| 合計時間 | 10時間 | 10.5時間 |
| 開発アプローチ | 段階的な手動指導 | **自動プロンプトシステム** |
| GitHub | 想定あり | **第1回で必須セットアップ** |
| 要件定義 | マニュアル作成 | **AIプロンプト自動生成** |
| DB/認証実装 | 段階的説明 | **MASTER_INTEGRATION_PROMPT使用** |
| プレゼン | なし | **第7回で実施** |

### コアコンセプト
1. **マインドセット変革**: 「プログラミング学習」から「コンテクストコントロール」へ
2. **IDE革命**: Cursor 2.0を中心とした開発環境
3. **自動化ワークフロー**: 4つの要件定義プロンプト + 統合実装プロンプト
4. **実践重視**: 最初から最後まで自分のプロダクト開発

#### 2. LLM基礎とプロンプトエンジニアリング (30分)
**企業版ソース**: `1-2_LLM基礎.md` (全体)

##### 2.1 LLMとは何か (10分)

**LLMの本質**:
- **Large Language Model**: 大規模言語モデル
- **仕組み**: 膨大なテキストデータから学習した「次の単語予測マシン」
- **できること**:
  - 文章生成
  - 翻訳
  - 要約
  - コード生成
  - 質疑応答

**主要LLM比較**:
| モデル | 開発元 | 特徴 | 料金 |
|--------|--------|------|------|
| GPT-4 | OpenAI | 最も有名、高性能 | 有料 |
| Claude 3.5 Sonnet | Anthropic | コード生成に強い | 有料 |
| Gemini | Google | マルチモーダル | 一部無料 |

##### 2.2 プロンプトエンジニアリングの基礎 (15分)

**"Text is KING" の原則**:
> LLMへの入力（プロンプト）の質が、出力の質を100%決定する

**良いプロンプトの5原則**:
1. **具体的に**: 「良いコードを書いて」→「Next.js 15のApp Routerを使って、ログイン画面を作成してください」
2. **文脈を与える**: 「あなたはReactの専門家です」から始める
3. **制約を明示**: 「TypeScriptを使用」「Tailwind CSSでスタイリング」
4. **出力形式を指定**: 「Markdown形式で」「コードブロックで」
5. **段階的に**: 複雑なタスクは小さく分割

**実演例** (Cursorでライブコーディング):
```
❌ 悪い例:
「ログイン画面を作って」

✅ 良い例:
「Next.js 15のApp Routerを使って、以下の要件を満たすログイン画面を作成してください:
- メールアドレスとパスワードの入力フォーム
- Tailwind CSS v4でスタイリング
- Server Actionsでフォーム送信
- バリデーションエラー表示機能
- TypeScriptで型安全に実装」
```

##### 2.3 Cursorでのプロンプト実践 (5分)

2. プロンプト入力

### D. トラブルシューティングFAQ

#### Q1: Cursorが遅い
**A**: API Key の制限に達している可能性 → プラン確認

#### Q2: Supabase でデータが保存されない
**A**: RLS ポリシー確認 → `service_role` key の設定確認

**A**: ビルドログ確認 → ローカルで `npm run build` → エラー修正

#### Q4: Clerk ログインができない
**A**: Vercel URL を Clerk の Allowed Domains に追加

#### Q5: 環境変数が反映されない
**A**: Vercel で再デプロイ（Deployments → ... → Redeploy）
