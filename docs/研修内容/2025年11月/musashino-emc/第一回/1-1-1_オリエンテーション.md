## 🎯 なぜ「プログラミング」を題材にするのか？

### **本質は「コンテクストコントロール」の習得**

**この講義のゴールは、プログラムを作ることではありません**

本当のゴールは：
> **AIから高品質な出力を引き出す「コンテクストコントロール」を身につけること**

### **なぜプログラミングが最高の教材なのか？**

#### **理由1: 出力品質を高める最高の訓練**

プログラムは、**一文字でもコードが間違っていればエラーを吐きます**

```
通常の文章作成:
- 多少の間違いがあっても意味は通じる
- 完璧でなくても成果物になる

プログラム開発:
- 一文字でも間違えば動かない
- 完璧に噛み合わないと成果にならない
```

**つまり**：
- プログラムを完成させるには、**相当な回数のAIとの対話**が必要
- **AIへの指示出し**と**コンテクストの渡し方**に熟達する必要がある

> **難易度が高い課題に挑戦することで、AIとの付き合い方とセンスが磨かれる**

---

#### **理由2: ビジネス文書は「余裕」になる**

プログラミングで基礎が身につけば、他のタスクは比較的容易です：

| タスク | 難易度 | 必要なスキル |
|--------|--------|------------|
| **プログラム開発** | ★★★★★ | 完璧なコンテクストコントロール |
| **事業計画書** | ★★★☆☆ | ハルシネーション排除、擦り合わせ |
| **プレゼン資料** | ★★☆☆☆ | 編集眼、ブラッシュアップ |
| **メール文章** | ★☆☆☆☆ | 基本的な指示 |

**一番難しい課題（プログラミング）をマスターすれば、他は簡単になります**

---

#### **理由3: 総合力が試される**

V0、Replit など、プロトタイプを作るツールはたくさんあります。

**でも、それでは不十分です**

学んでほしいのは：
- ✅ **一撃で自分の意にそぐったものを出させる**コンテクストの渡し方
- ✅ 思っていることを伝える**言語化力**
- ✅ それをブーストさせる**AIの使い方**
- ✅ 普段からの**ドキュメント(コンテクスト)管理の仕方**

> プログラミングという難題に挑戦することで、これらの総合力が身につきます

## 🤖 IDE型AIとは何か？

### **なぜIDEを学ぶべきなのか**

**IDE（統合開発環境）型AI**とは：
- Cursorのような、開発環境に統合されたAI
- ファイル構造、プロジェクト全体を理解
- コード生成、修正、リファクタリングを一気通貫でサポート

### **チャット型AIとの決定的な違い**

| 項目 | チャット型AI<br/>（ChatGPT, Claude等） | IDE型AI<br/>（Cursor等） |
|------|-------------------------------------|------------------------|
| **理解範囲** | 1つの会話 | プロジェクト全体 |
| **コード反映** | コピペが必要 | 自動でファイルに反映 |
| **効率性** | 低い（手作業多い） | 高い（自動化） |
| **用途** | アイデア出し、学習 | **本格的な開発** |

### **なぜIDEが重要なのか**

> チャット型AIだけでは、**「勉強」はできても「開発」はできません**

- 実際のプロダクト開発には、IDE型AIが必須
- この講義では、**Cursor**を使って本格開発を学びます


**重要**：
> 横着せず、**具体的に、詳しく、明確に**指示を出す癖をつけましょう

---

### **2️⃣ AIの出力を一言一句、読み漏らさない**

**これができない人が、実は圧倒的に多いです**

AI開発でエラーが起きる最大の原因：
- ❌ AIの説明を読み飛ばす
- ❌ 「よくわからないこと書いてある」と無視する
- ❌ AIが「やってください」と言ったことをやらない

**例えば**：
```
AI: 「.env.local ファイルを作成し、以下の内容を記載してください：
     NEXT_PUBLIC_SUPABASE_URL=your_url_here
     NEXT_PUBLIC_SUPABASE_ANON_KEY=your_key_here」

学生A: よくわからないから飛ばそう → エラー発生
学生B: 言われた通りファイル作成 → 正常動作
```

**ソフトウェア開発では、AIの指示を正確に実行することが重要です**

特に：
- ✅ アカウント作成の指示
- ✅ ファイルの作成・編集の指示
- ✅ 環境変数の設定

これらはAIが「セキュリティ上の理由で自動化できない場合がある」ため、人間がやる必要があります。

**徹底的に読む癖をつけてください**

---

### **3️⃣ まず自分で調べる、AIに聞く、それでもダメなら講師に聞く**

**このルールを徹底します**

```
質問の優先順位：
1. まず自分で考える（3分）
2. ChatGPT/Claude で検索・質問（5分）
3. AIの回答を実際に試す（10分）
4. それでも解決しないなら講師に質問

❌ NG: いきなり講師に「できません」
✅ OK: 「AIにこう聞いて、こう試したけど、〇〇がうまくいきません」
```

**なぜこのルールがあるのか？**

1. **AIに聞く癖がつく**
   - 卒業後も自分で問題解決できる

2. **対話の回数が増える**
   - AIとの対話が増えるほど、選別眼が身につく
   - 「これは正しい」「これは嘘だった」が分かるようになる

3. **実践力がつく**
   - 情報を集め、試し、自分の糧にする力

**日常生活でも実践しましょう**：
> 「これ、AIに作らせられないかな？」と考える癖をつける

## 💡 コンテクストコントロールとは

### **Vibe Coderの最重要スキル**

**コンテクストコントロール**とは：

> AIに対して、**適切な情報（コンテクスト）を与え、望む結果を引き出す技術**

### **コンテクストコントロールの3要素**

1. **What（何を作るか）**
   - 明確な要件定義
   - 具体的なゴール設定

2. **How（どう伝えるか）**
   - プロンプトの書き方
   - 情報の構造化

3. **Why（なぜそうするか）**
   - ビジネスロジックの理解
   - ユーザー視点

### **例：良いプロンプト vs 悪いプロンプト**

❌ **悪いプロンプト**：
```
SNSを作って
```

✅ **良いプロンプト**：
```
Next.js 15 で、以下の機能を持つSNSを作成してください：

1. 投稿機能
   - テキスト入力（最大280文字）
   - 画像アップロード（最大4枚）
   - 投稿日時の表示

2. タイムライン
   - 最新の投稿から順に表示
   - 無限スクロール

3. UI要件
   - Tailwind CSS でモダンなデザイン
   - スマホ対応（レスポンシブ）

技術スタック: Next.js 15, TypeScript, Tailwind CSS
```

## 📝 今日の宿題

### **宿題：アイデアのブラッシュアップと体験**

**タスク1: Cursorでアイデアをブラッシュアップ（必須）**

1. **自分が作りたいアプリのテーマを考える**
   - 例：音楽好き向けSNS、ペット自慢SNS、読書記録SNS、タスク管理アプリなど
   - 何でもOK！自分が欲しいと思うものを考えてください

2. **Cursorのチャット機能でアイデアを対話**
   - Cursorを開き、チャット機能を起動
     - **Mac**: `Cmd+L`
     - **Windows**: `Ctrl+L`
   - 「こういうアプリを作りたい」とざっくり伝える
   - AIから質問されたら答える
   - 対話を通じてアイデアを具体化する
   - **要件定義書は作らなくてOK**（次回以降で詳しく学びます）

3. **対話の記録をMarkdown形式で保存**
   - 対話が終わったら、Cursorのチャットに「この対話内容をMarkdown形式で書き出して」と指示
   - 生成されたMarkdownを確認したら、プレビューで見やすく表示
     - **Mac**: `Cmd+Shift+V`（Markdownプレビュー）
     - **Windows**: `Ctrl+Shift+V`
   - `idea.md` などのファイル名で保存
   - 保存方法がわからない場合は、Cursorに「Markdownファイルの保存方法を教えて」と聞いてみてください
   - 次回授業で共有します

> **ポイント**: CursorはIDE型AIツールなので、チャット型AI（ChatGPT、Claude）とは違う体験ができます。まずはCursorに慣れてみましょう！

**よく使うCursorショートカット（Mac）**:
- `Cmd+L`: チャット起動
- `Cmd+I`: Composer起動（コード生成・編集）
- `Cmd+K`: インライン編集（選択部分を修正）
- `Cmd+Shift+V`: Markdownプレビュー
- `Cmd+Shift+P`: コマンドパレット
- `Control+C`: ターミナルでコマンドをキャンセル（よく使う！）
