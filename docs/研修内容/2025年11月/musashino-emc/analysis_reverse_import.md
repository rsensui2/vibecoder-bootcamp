### 3. **宿題・課題のセクションが具体的で実行しやすい**

#### 学生向けの宿題（非常に明確）
```markdown
**タスク1: Cursorでアイデアをブラッシュアップ（必須）**

1. **自分が作りたいアプリのテーマを考える**
   - 例：音楽好き向けSNS、ペット自慢SNS、読書記録SNS、タスク管理アプリなど
   - 何でもOK！自分が欲しいと思うものを考えてください

2. **Cursorのチャット機能でアイデアを対話**
   - Cursorを開き、チャット機能を起動
     - **Mac**: `Cmd+L`
     - **Windows**: `Ctrl+L`
   - 「こういうアプリを作りたい」とざっくり伝える
   - AIから質問されたら答える
   - 対話を通じてアイデアを具体化する
   - **要件定義書は作らなくてOK**（次回以降で詳しく学びます）

3. **対話の記録をMarkdown形式で保存**
   - 対話が終わったら、Cursorのチャットに「この対話内容をMarkdown形式で書き出して」と指示
   - 生成されたMarkdownを確認したら、プレビューで見やすく表示
     - **Mac**: `Cmd+Shift+V`（Markdownプレビュー）
     - **Windows**: `Ctrl+Shift+V`
   - `idea.md` などのファイル名で保存
   - 保存方法がわからない場合は、Cursorに「Markdownファイルの保存方法を教えて」と聞いてみてください
   - 次回授業で共有します

> **ポイント**: CursorはIDE型AIツールなので、チャット型AI（ChatGPT、Claude）とは違う体験ができます。まずはCursorに慣れてみましょう！

**よく使うCursorショートカット（Mac）**:
- `Cmd+L`: チャット起動
- `Cmd+I`: Composer起動（コード生成・編集）
- `Cmd+K`: インライン編集（選択部分を修正）
- `Cmd+Shift+V`: Markdownプレビュー
- `Cmd+Shift+P`: コマンドパレット
- `Control+C`: ターミナルでコマンドをキャンセル（よく使う！）
```

### 4. **「この講義への取り組み方」セクション（重要★）**

#### 学生向けの警告とモチベーション喚起
```markdown
### **宿題は必ずやってください**

この講義では、ほぼ毎回宿題を出します。

**宿題をやらないと、次の回についていけません。これは脅しではなく、事実です。**

なぜなら、AI駆動開発は「積み重ね」のスキルだからです。
- 第1回の宿題をやらない → 第2回の内容が理解できない
- 第2回の宿題をやらない → 第3回の実装ができない
- という形で、どんどん差が開いていきます。

### **この講義のペース**

- 全7回の講義（来年1月まで、講義がない週もあります）
- 毎回90分の講義
- 毎回2〜3時間程度の宿題
- 講義がある週は合計5〜6時間のコミットメントが必要です

**忙しいのは分かります。でも、だからこそ、覚悟を決めて参加してください。**

この7回の講義を完走した先には、
- **組織で希少な「アウトプットを出せる人材」になれるスキル**
- **ビジネスで圧倒的に差別化できる実績**
- **自分のアイデアを形にできる力**

が待っています。
```

**逆輸入提案**：
- 企業向けにも「課題の重要性」を明示
- 「忙しい中でも時間を確保する覚悟」を促す

### 5. **「鍛える3つのスキル」セクションが超わかりやすい**

```markdown
## ⚡ この講義で徹底的に鍛える3つのスキル

### **1️⃣ 質の高いインプットを横着せずに与える**

**AIは、あなたが与えた情報の質でしか答えられません**

❌ **悪い例**：
```
SNSを作って
```

✅ **良い例**：
```
Next.js 15 で音楽好き向けSNSを作成してください。

主要機能：
1. 投稿機能（好きな曲の共有、最大280文字）
2. いいね・コメント機能
3. ユーザープロフィール

デザイン：モダンで音楽を感じさせる雰囲気
```

**重要**：
> 横着せず、**具体的に、詳しく、明確に**指示を出す癖をつけましょう

---

### **2️⃣ AIの出力を一言一句、読み漏らさない**

**これができない人が、実は圧倒的に多いです**

AI開発でエラーが起きる最大の原因：
- ❌ AIの説明を読み飛ばす
- ❌ 「よくわからないこと書いてある」と無視する
- ❌ AIが「やってください」と言ったことをやらない

**例えば**：
```
AI: 「.env.local ファイルを作成し、以下の内容を記載してください：
     NEXT_PUBLIC_SUPABASE_URL=your_url_here
     NEXT_PUBLIC_SUPABASE_ANON_KEY=your_key_here」

学生A: よくわからないから飛ばそう → エラー発生
学生B: 言われた通りファイル作成 → 正常動作
```

**ソフトウェア開発では、AIの指示を正確に実行することが重要です**

特に：
- ✅ アカウント作成の指示
- ✅ ファイルの作成・編集の指示
- ✅ 環境変数の設定

これらはAIが「セキュリティ上の理由で自動化できない場合がある」ため、人間がやる必要があります。

**徹底的に読む癖をつけてください**

### **3️⃣ まず自分で調べる、AIに聞く、それでもダメなら講師に聞く**

**このルールを徹底します**

```
質問の優先順位：
1. まず自分で考える（3分）
2. ChatGPT/Claude で検索・質問（5分）
3. AIの回答を実際に試す（10分）
4. それでも解決しないなら講師に質問

❌ NG: いきなり講師に「できません」
✅ OK: 「AIにこう聞いて、こう試したけど、〇〇がうまくいきません」
```

**なぜこのルールがあるのか？**

1. **AIに聞く癖がつく**
   - 卒業後も自分で問題解決できる

2. **対話の回数が増える**
   - AIとの対話が増えるほど、選別眼が身につく
   - 「これは正しい」「これは嘘だった」が分かるようになる

3. **実践力がつく**
   - 情報を集め、試し、自分の糧にする力

**日常生活でも実践しましょう**：
> 「これ、AIに作らせられないかな？」と考える癖をつける
```

**逆輸入提案**：
- 企業向けにも「3つのスキル」セクションをそのまま追加
- 企業での具体例に置き換え（例：「会議資料を作って」→ 詳細な指示）
- 「質問の優先順位」は組織文化として導入を推奨

### 6. **「あなた自身の価値：AIを凌駕するために」セクション**

#### 学生向けの編集眼の説明
```markdown
## 🎨 あなた自身の価値：AIを凌駕するために

### **AIの出力は「たたき台」でしかない**

AIがそのまま出力したものは、**誰が見ても「AIで作ったな」と分かります**

本当に価値があるのは：
- ✅ **あなたの経験**から生まれるアイデア
- ✅ **あなたの視点**で編集された内容
- ✅ **あなたの課題意識**が反映されたプロダクト

### **編集者としてのあなた**

```
AIの役割：素材を提供する料理人
あなたの役割：どんな料理にするか決めるシェフ

❌ AIが作ったものをそのまま使う
   → つまらない、誰でも作れるものになる

✅ AIの出力を、あなたの編集眼でブラッシュアップ
   → あなたにしか作れない、価値あるものになる
```

### **プログラミングでは、まずAIの言うことを聞く**

ただし、文章やデザインと違い、**プログラミングに関しては最初はAIの言うことを素直に聞きましょう**

理由：
1. 皆さんはまだ知識が浅い
2. AIの提案を実際に試すことで学べる
3. 徐々に「なぜこうするのか」が分かってくる

**最終ゴール**：
> AIを利用しながら学び、**いずれはAIを凌駕する存在**になる

このスタンスで7週間、取り組んでいきましょう。
```

**逆輸入提案**：
- 企業向けにも「編集眼」の重要性を強調
- 「AIの出力をそのまま使うリスク」を明示
- 「ビジネスパーソンとしての価値発揮」という視点を追加
