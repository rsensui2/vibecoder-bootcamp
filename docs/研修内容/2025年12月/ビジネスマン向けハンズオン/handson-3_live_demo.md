# ライブデモ: 企画書→LP→説明資料生成

## デモシナリオ概要

**題材**: 新規事業『MeetingMind』
- サービス内容: ビジネスマン向け、AIを活用した議事録自動作成・タスク抽出サービス
- 目的: 1つのアイデアから、企画書・LP・顧客向け資料・投資家向け資料を一気通貫で生成
- 所要時間: 20分（企画書5分、LP 7分、説明資料8分）

**デモの目的**:
1. **コンテクストの一貫性**: 同じコンテクストから複数のアウトプットが生成される
2. **スピード**: 従来数週間かかる作業が20分で完了
3. **品質**: AIが構造化された高品質なアウトプットを生成
4. **ファイル自動生成**: 次々とファイルが作られる様子を体験

## Phase 1: 企画書生成（5分）

### 実演内容

**使用ツール**: Antigravity/Cursor
**アクション**: **音声入力でアイデアをばーっと喋る**、AIが企画書を生成

**音声入力の威力**:
- タイピングより圧倒的に速い
- アイデアを自然に言語化できる
- 思考の流れをそのままAIに渡せる

**実演方法**:
1. プロジェクトフォルダを作成
2. Antigravity/Cursorのチャット欄を開く
3. **音声入力ボタンを押して、以下の内容をばーっと喋る**
4. AIが企画書ファイルを自動生成

### プロンプト（コピペ用・または音声入力）

```
【新規事業企画書作成 - MeetingMind】

事業アイデア:
「ビジネスマン向け、AIを活用した議事録自動作成・タスク抽出サービス『MeetingMind』」

背景:
- 会議後の議事録作成に平均30分かかる
- タスクの抜け漏れが発生
- 議事録の質が担当者によってバラつく

企画書を5つのMarkdownファイルに分けて作成してください:

1. executive_summary.md（エグゼクティブサマリー）
   - 事業概要
   - 市場機会
   - 競争優位性

2. market_analysis.md（市場分析）
   - ターゲット市場
   - 顧客の課題
   - 市場トレンド

3. solution_business_model.md（ソリューションと収益モデル）
   - 提供する価値
   - 主な機能3つ
   - 料金プラン3つ
   - 収益予測

4. competition_roadmap.md（競合分析とロードマップ）
   - 主要競合3社
   - 差別化ポイント
   - 3ヶ月の実装計画

5. resources_risks.md（リソースとリスク）
   - 必要な体制
   - 初期投資
   - リスクと対策

各ファイル100-200行くらいで、投資家向けのトーンで。
```

### 実演ポイント

1. **プロンプト投稿**: 上記プロンプトをコピペ（または音声で指示）
2. **AIの作業過程を見せる**: 5つのファイルが次々と生成される様子
3. **ファイル確認**: 5つのMarkdownファイルが並ぶ
4. **内容確認**: executive_summary.md を開いて内容を軽く確認

### 強調すべきポイント

- **5つのファイルが一度に作られる**: これがIDE型AIの威力
- **構造化されている**: 各ファイルが明確な役割を持つ
- **100-200行ずつ**: 読みやすく、編集しやすいサイズ
- **コンテクストとして保存される**: この5ファイルがLP生成の材料になる
- **NotebookLMで活用できる**: 後でスライド作成に使える

## Phase 2: LP（ランディングページ）生成（7分）

### 実演内容

**使用ツール**: Antigravity/Cursor（同じセッション内で継続）
**アクション**: 上記企画書をベースにLPを生成（**音声入力でも可**）

### プロンプト（コピペ用・または音声入力）

```
【上記企画書をもとにランディングページを3パターン作成】

上記で作成した MeetingMind の企画書をベースに、コンバージョン重視のランディングページを**3つの異なるデザインパターン**で作成してください。

## 共通の構成:
1. ヒーローセクション
   - キャッチコピー（インパクト重視）
   - サブコピー（ベネフィット明確化）
   - CTA（無料トライアル申込ボタン）
   - ビジュアル要素

2. 課題提起セクション
   - 「こんな悩みありませんか?」
   - 3つの具体的なペインポイント
   - 共感を生む表現

3. ソリューション紹介
   - MeetingMindが解決すること
   - Before/After比較
   - 数値で示す効果

4. 主要機能紹介（3つ）
   - 機能1: AI議事録自動作成
   - 機能2: タスク自動抽出
   - 機能3: 過去議事録の横断検索
   - それぞれアイコン＋説明＋ベネフィット

5. 利用者の声（架空でOK）
   - 3名の利用者コメント
   - 職種・課題・効果を含む

6. 料金プラン
   - フリープラン
   - ビジネスプラン（推奨）
   - エンタープライズプラン
   - 機能比較表

7. FAQ
   - よくある質問5つ

8. CTA（最終訴求）
   - 「今すぐ無料で始める」ボタン

9. お問い合わせフォーム
   - 名前、メール、会社名、問い合わせ内容

## デザインパターン（3つ作成）:

### パターンA: モダン・ミニマル
- カラー: ブルー＆ホワイト（#2563eb, #ffffff）
- スタイル: シンプル、余白たっぷり、幾何学的要素
- フォント: サンセリフ、大きめの見出し
- 特徴: カード型レイアウト、グラデーション背景、シャドウ効果

### パターンB: ビジネス・プロフェッショナル
- カラー: ダークブルー＆ゴールド（#1e3a8a, #f59e0b）
- スタイル: 高級感、信頼性重視
- フォント: セリフ＋サンセリフの組み合わせ
- 特徴: 2カラムレイアウト、区切り線、統計数値の強調表示

### パターンC: カラフル・モダン
- カラー: マルチカラー（ブルー、グリーン、パープルのグラデーション）
- スタイル: 活気ある、イノベーティブ
- フォント: 太字のサンセリフ
- 特徴: フルスクリーンセクション、アニメーション効果、大胆な配色

## 技術要件:
- **HTML5 + CSS3のみ**（フレームワーク不使用）
- レスポンシブ対応必須（モバイルファースト）
- CSS Grid / Flexboxを活用
- 軽量なアニメーション（CSS transitionのみ）
- アイコンはUnicode絵文字またはSVGで実装

## ファイル構成:
```
lp/
├── pattern-a/
│   ├── index.html
│   └── style.css
├── pattern-b/
│   ├── index.html
│   └── style.css
└── pattern-c/
    ├── index.html
    └── style.css
```

## デザイン品質要件:
- **凝ったデザイン**: プロのデザイナーが作ったと思えるクオリティ
- **視覚的階層**: 情報の優先順位が明確
- **ホバーエフェクト**: ボタンやカードに洗練されたインタラクション
- **タイポグラフィ**: 読みやすく美しいフォント設定
- **カラーハーモニー**: 配色理論に基づいた調和
- **スペーシング**: 適切な余白とリズム

各パターンで完全に動作する、プロフェッショナルなランディングページを作成してください。
```

### 実演ポイント

1. **プロンプト投稿**: 上記プロンプトをコピペ（または音声で指示）
2. **AIの作業過程を見せる**:
   - `lp/` ディレクトリと3つのサブディレクトリを作成
   - 6つのファイル（3パターン × 2ファイル）を次々と生成
   - 各パターンで異なるデザインが作られる様子
3. **ファイル一覧を確認**: `lp/pattern-a/`, `lp/pattern-b/`, `lp/pattern-c/` 配下のファイルが並ぶ
4. **ブラウザでプレビュー**:
   - 各HTMLファイルを直接ブラウザで開く
   - 3つのデザインパターンを並べて表示
   - デザインの違いを視覚的に比較
5. **デザイン確認**:
   - パターンAのミニマルなデザイン
   - パターンBの高級感あるデザイン
   - パターンCのカラフルなデザイン
   - レスポンシブ動作（ブラウザ幅を変えて確認）

### 強調すべきポイント

- **6つのファイルが一度に作られる**: これがIDE型AIの威力
- **3つの異なるデザイン**: AIがそれぞれ異なるスタイルで作成
- **企画書との一貫性**: 同じコンテクストから生成されているため内容が一致
- **即座にプレビュー**: HTMLファイルをブラウザで開くだけ、ビルド不要
- **デザインの多様性**: 「パターンAをもっとシンプルに」「パターンBの色を変えて」など、個別修正も可能
- **フレームワーク不要**: HTML + CSSだけでプロレベルのデザインが作れる

## Phase 3: NotebookLMでスライド作成（8分）

### 実演内容

**使用ツール**: NotebookLM（https://notebooklm.google.com/）
**アクション**: Phase 1で作成した5つのMarkdownファイルをNotebookLMに読み込ませ、AIにスライドを作成させる

### ステップ1: NotebookLMを開く（1分）

1. ブラウザで **NotebookLM**（https://notebooklm.google.com/）にアクセス
2. 新しいノートブックを作成
3. 画面を参加者に見せながら操作

### ステップ2: 企画書ファイルをソースとして追加（2分）

**実演**:
1. NotebookLMの「ソースを追加」ボタンをクリック
2. Phase 1で作成した5つのMarkdownファイルを**ドラッグ&ドロップ**:
   - `executive_summary.md`
   - `market_analysis.md`
   - `solution_business_model.md`
   - `competition_roadmap.md`
   - `resources_risks.md`

3. 5つのファイルがソースとして読み込まれたことを確認

**強調ポイント**:
- **複数ファイルをまとめてソースにできる**: NotebookLMの強力な機能
- **コンテクスト全体を理解**: 5つのファイル全体から情報を引き出せる

### ステップ3: 顧客向けスライド作成を指示（2分・音声入力推奨）

**プロンプト**（音声入力または入力）:

```
この企画書をもとに、顧客向けのサービス紹介プレゼンスライドを作ってください。

対象: 企業の情報システム部門・総務部門の担当者

20枚くらいで:
- 表紙
- 課題提起（現状の会議運営の課題）
- ソリューション紹介（MeetingMindとは）
- 主な機能3つ（それぞれ詳しく）
- 導入効果（時間削減、コスト削減）
- 料金プラン
- 導入事例
- FAQ
- お問い合わせ

わかりやすく、親しみやすいトーンで。
```

**実演**:
- NotebookLMがスライド形式で生成する様子を見せる
- 生成されたスライドを確認
- 必要に応じて「もっと具体的に」「図解を増やして」などの追加指示

### ステップ4: Google Slidesにエクスポート（1分・オプション）

**実演**:
1. NotebookLMの「エクスポート」ボタンをクリック
2. 「Google Slidesにエクスポート」を選択
3. Google Slidesで開いて編集可能なことを示す

### ステップ5: 投資家向けスライド作成（2分・時間があれば）

**プロンプト**（音声入力または入力）:

```
同じ企画書をもとに、今度は投資家向けのピッチ資料を作ってください。

対象: VC、エンジェル投資家
目的: シリーズA資金調達（3億円）

15枚で:
- ビジョン
- 課題と市場機会
- ソリューション
- ビジネスモデル
- トラクション
- 競合優位性
- チーム
- ロードマップ
- 資金使途

データドリブンで、説得力のあるトーンで。
```

**実演**:
- 同じソースから、異なるオーディエンス向けの資料が作られることを見せる
- トーンの違いを確認

### 実演ポイント

1. **ドラッグ&ドロップの簡単さ**:
   - ファイル選択→ドラッグ→完了
   - Markdown構文を気にする必要なし

2. **複数ファイルをソースに**:
   - 5つのファイル全体のコンテクストを理解
   - 情報の一貫性が自動的に保たれる

3. **音声入力の活用**:
   - プロンプトをタイプせず、話すだけでOK
   - より自然に、速く指示できる

4. **即座にスライド生成**:
   - NotebookLMがスライド形式で出力
   - Google Slidesで編集可能

5. **同じソース、異なるオーディエンス**:
   - 顧客向け: 親しみやすく、わかりやすく
   - 投資家向け: データドリブン、説得力重視
   - AIが自動的にトーンを調整

### 強調すべきポイント

- **NotebookLMの威力**: 複数のMarkdownファイルを一度に読み込み、全体を理解してスライド作成
- **コンテクストの蓄積と循環**: Phase 1で作った企画書 → Phase 3でスライド生成（コンテクストが循環）
- **エクスポートの柔軟性**: Google Slidesで編集・プレゼン可能
- **従来の作業量**: 企画書作成→スライド作成で数週間 → **今日: 20分**
- **音声入力の活用**: タイピングせずに話すだけでスライド作成指示

## デモ全体の振り返り（デモ後に話す内容）

### 今日見たこと

1. **1つのアイデア** → **11個のファイル**（企画書5ファイル + LP 6ファイル）+ **NotebookLMでスライド作成**
2. **20分** → 従来なら数週間の作業
3. **コンテクストの蓄積と循環** → 企画書5ファイル → LP 3パターン → NotebookLMでスライド
4. **ファイル自動生成** → 次々とファイルが作られる様子
5. **ツールの連携** → Antigravity/Cursor（ファイル生成）+ NotebookLM（スライド作成）

### ChatGPTとの決定的な違い

| 特徴 | ChatGPT（チャット型） | Antigravity/Cursor（IDE型） |
|------|---------------------|--------------------------|
| 出力形式 | テキストのみ | **複数ファイル自動生成** |
| コンテクスト | 会話単位 | **プロジェクト全体** |
| 編集 | コピペして手動編集 | **その場で即座に修正** |
| プレビュー | 不可 | **ローカルで即座に確認** |
| 複雑な構造 | 難しい | **ディレクトリ構造も自動作成** |

### アウトプット革命の本質

**Before（従来）**:
- アイデア → 議事録の1行で終了
- 企画書 → リソース不足で頓挫
- プロトタイプ → 外注で数ヶ月＋数十万円

**After（Vibe Coder）**:
- アイデア → 20分で企画書・LP・資料完成
- 即座に検証可能
- 高速反復で成長
