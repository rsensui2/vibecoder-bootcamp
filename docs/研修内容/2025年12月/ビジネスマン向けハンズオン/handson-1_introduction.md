# 自己紹介 + VibeCoder Bootcamp紹介

## 話す内容

### 1. 講師紹介

**自己紹介の核心メッセージ**:
> 「2022年12月まで、コードを1行も書けませんでした。今も書けません」

**経歴**:

この10年くらいで私と出会った方は、いつ出会ったかによって私が何をやっているかが変わっているかと思います。

- **Fuji Xerox（現富士フイルムビジネスイノベーション）**: 新卒で入社、3年間営業を経験
- **MAMORIO（現NEARIZE）**: 2014年、落とし物防止タグのスタートアップに3人目のメンバーとして入社、3ヶ月後にCOOに就任。IoTの世界で活動
- **TEKION Group（テキオングループ）**: 2020年設立。完全に私1人の会社で、グループという名前ですが私1人です
- **CyberneX**: ブレインコンピューターインターフェース（イヤホン型の脳波計など）を作っている会社のCSOとして5年間活動中。現在も本業として継続中

基本的にCOOやCSOなどビジネスサイドを担当することが多く、**エンジニアリングのバックグラウンドは全くありません**。営業バックグラウンドです。

**開発実績**:

しかし、2022年末にChatGPTが登場してから、この2〜3年の間に**約80個のアプリを開発**しています。ただし、今もコード1行も書けません。書けないのにAIとの対話だけで80本アプリを作っており、その一部は月で500万円ほどの売上を立てているものもあります。

**市場価値の実証**:

去年の夏前から1年くらいほぼフリーターみたいな感じで、1日中Vibe Codingやっていました。今年の頭、ある受託開発の会社の社長にお願いして、**1ヶ月間期間限定で雇ってもらった**んです。

社員の人には言わずに、「こいつちょっとエンジニア志望してるからどんなもんか査定して欲しい」ということで1ヶ月働かせてもらいました。1ヶ月後、その会社の通常のエンジニア評価手法で行った時、私はAIしか使っていない、全部の回答をAIに頼りましたという状態で仕事をしたら、**月収160万円**という査定をいただきました。

文系で普通に営業活動などをして月160万円という会社はあまりないと思いますが、やべえなと思いました。そこまでAIが来ていると。

**なぜVibe Codingを始めたのか**:

スタートアップをやっていて思うのは、**アイデアだけでは意味がない**ということです。世の中に自分の考えたものを物として出して通るということが重要です。アウトプットを出すことが最も重要だと思っています。

ただ、特にプロダクトという形になると、私はずっと課題だったのが「物づくりの能力がない」ということでした。マーケットのことを考える時間も長いですし、お客さんとも接するので、アイデアは色々出てくるんですが、エンジニアさんにお願いしないと物が作れませんでした。

それが、この技術を使って「あれ、何でも作れるじゃん」という思考回路に変わってきました。スピードも上がっています。これが私の中ですごく革命的なことでした。

### 2. VibeCoder Bootcampの実績

**プログラム概要**:
- **期間**: 1.5ヶ月で企画→開発→デプロイまで一貫して習得
- **受講満足度**: 100%（2025年12月時点、全受講者アンケート結果）
- **学習形式**: 伴走型サポートで挫折させない設計

**受講者層の特徴**:

最近は上場企業やベンチャー、あるいは中小企業経営者（CEO）の方にマンツーマンでレッスンをさせていただいており、ある種金銭的には本業になってきました。

- 上場企業の経営者・役員
- ベンチャー企業の創業者・CTO以外の経営陣
- 中小企業の経営者・事業責任者
- 各部署のリーダー・マネージャー

**企業導入の戦略的価値**:

会議やSlackで「いいね」だけついて終わる、議事録には残っているけど誰も巻き取らないということがあると思います。世界的にVibe Codingでやっている人たちがなぜあんなに熱中しているのかというと、**眠らせてこざるを得なかったアイデアが、大体形的なものであれば形になっちゃう**というところです。

- 「各部署に1人Vibe Coderを配置する」企業戦略として導入
- 組織全体のスピード感が激変した実例多数
- **AI使える人間が各部署に数人いる会社は、動きが圧倒的に速い**

**投資対効果**:
- 外注開発費の削減: 30万円の案件を自作で対応 → 25万円以上の直接節約
- 新規売上の創出: 浮いた時間で副業開発、月30万円の追加収入
- キャリア価値向上: 転職市場で年収数百万円アップのオファー事例
- 意思決定の早期化: アイデアを「作って触って」判断できる

**なぜ満足度100%なのか**:
1. **非エンジニア出身講師による設計**: 躓きポイントを完全に理解
2. **実践重視**: 座学より「作りながら学ぶ」アプローチ
3. **ビジネスインパクト直結**: 学んだその日から業務に活用可能
4. **伴走型サポート**: 個別の課題に対応、挫折させない

### 3. 今日のゴール

**ワークショップの目的**:
> 「アイデアの墓場」から「アウトプットの工場」への転換を体験する

**Before（従来の現実）**:
- 素晴らしいアイデアが出る → 議事録の1行で終了
- 企画書を書く → リソース不足で頓挫
- プロトタイプが必要 → 外注で数ヶ月＋数十万円

**After（今日体験すること）**:
- アイデアを入力 → 60分で企画書・LP・説明資料が完成
- その場で形にして検証可能
- 失敗してもコストは実質ゼロ、すぐ次のアイデアへ

**今日持ち帰っていただくもの**:
1. **体験**: ファイルが次々と作られる様子を目撃、自分のアイデアで実践成功
2. **知識**: Vibe Codingの本質理解、IDE型AIの威力
3. **マインドセット**: 「できない」を「まず作ってみる」に変える

### 4. 参加者への期待

**今日のスタンス**:
- 完璧を求めない → まず作ってみる
- 技術的理解は不要 → ビジネス価値に集中
- 失敗を恐れない → エラーも学びの一部

**持ち帰って実践してほしいこと**:

Cursorを入れて、明日の朝から仕事する時、まずメールを書かせるでも何でもいいんで、とりあえず**マークダウンで吐き出せ**って言って使い始めることが第一歩だと思います。

それをベースに「あれもできないの？」「これができないの？」ともうAIに聞いてみてください。それが多分一番やるかやらないかの差がそこにあって、そこはちょっとでもやったことある人は、もういくらでもあとはこだわりだけの世界になってくるんです。

1. **今日使うツール（Antigravity/Cursor）を継続使用**
   - 無料プランでも十分に実践可能
   - 業務の小さな課題から試してみる

2. **「まず作ってみる」文化の醸成**
   - 会議で出たアイデアを、その日のうちに形にしてみる
   - 企画書だけでなく、動くプロトタイプを持ち込む

3. **組織への展開を検討**
   - 各部署に1人Vibe Coderがいるだけで、スピード感が変わる
   - VibeCoder Bootcampの導入を検討（会社の経費・助成金対象）

**重要なメッセージ**:
> 「木を植える最良の時期は20年前だった。次に良い時期は、今だ。」

**AI時代の組織戦略**:
- AI使える人間が各部署に数人いる会社は、動きが圧倒的に速い
- それを日本の企業に、スタートアップにもベンチャーにも大企業にも広げていきたい

**Welcome to the Revolution. You are the Revolution.**
