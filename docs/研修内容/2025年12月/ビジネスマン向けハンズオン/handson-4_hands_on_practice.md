# 参加者演習: 自分のアイデアで実践

## 演習の目的

- 自分の業務や興味のあるテーマで、同じフローを実践する
- 「ファイルが次々と作られる」体験を自分の手で実現する
- アイデアから具体的なアウトプットを生み出す成功体験を得る

## タスク説明（2分）

### 今からやっていただくこと

先ほどのデモと同じように、以下の3ステップを実践していただきます:

1. **企画書生成**（5分）
2. **LP生成**（5分）
3. **説明資料生成**（5分）

合計15分で、残り5分はトラブルシューティングと成果共有の時間です。

### アイデアの選び方

**重要**: 完璧なアイデアである必要はありません。以下のいずれかを選んでください:

#### パターンA: 業務改善系
- 営業日報自動集計ツール
- 社内FAQ自動生成システム
- 顧客問い合わせ対応支援ツール
- 勤怠管理＋工数分析システム
- 契約書レビュー支援ツール

#### パターンB: 新規事業系
- 地域密着型コミュニティアプリ
- 趣味特化型マッチングサービス
- オンライン学習プラットフォーム
- サブスク型〇〇配送サービス
- 専門家マッチングプラットフォーム

#### パターンC: 個人プロジェクト系
- 読書記録・分析アプリ
- 家計簿＋投資分析ツール
- 健康管理・食事記録アプリ
- ペット管理・健康記録アプリ
- 趣味のコレクション管理ツール

**迷ったら**: 「営業日報自動集計ツール」で進めてください（テンプレート用意あり）

## 演習手順

### ステップ1: アイデアを1つ選ぶ（2分）

**重要**: 完璧なアイデアは不要です。**とにかくさっさとAIに指示を投げる**ことが目的です。

**選び方**:
- **自分のアイデアがある人**: そのまま使ってください
- **迷っている人**: 下記のリストから1つ選んで、少しだけ自分ならではの要素を加えてください

**アイデアリスト**:
- 営業日報自動集計ツール
- 社内FAQ自動生成システム
- 顧客問い合わせ対応支援ツール
- 地域密着型コミュニティアプリ
- 読書記録・分析アプリ
- 家計簿＋投資分析ツール

### ステップ2: 企画書を作ってもらう（5分）

**使用ツール**: Antigravity または Cursor

**AIに伝える内容**（音声入力でも、タイプでもOK）:

```
【こんなテーマで新規事業の企画書を作ってください】

テーマ: 「〇〇〇〇」（あなたのアイデア）

企画書を5つのMarkdownファイルに分けて作ってください:

1. executive_summary.md（エグゼクティブサマリー）
   - 事業概要
   - 市場機会
   - 競争優位性

2. market_analysis.md（市場分析）
   - ターゲット市場
   - 顧客の課題
   - 市場トレンド

3. solution_business_model.md（ソリューションと収益モデル）
   - 提供する価値
   - 主な機能3つ
   - 料金プラン3つ

4. competition_roadmap.md（競合分析とロードマップ）
   - 主要競合3社
   - 差別化ポイント
   - 3ヶ月の実装計画

5. resources_risks.md（リソースとリスク）
   - 必要な体制
   - 初期投資
   - リスクと対策

各ファイル100-200行くらいで、投資家向けのトーンで。
```

**ポイント**:
- 上記の内容を**自分の言葉で**AIに伝えてください
- コピペ不要、**音声で話しかけるのが最速**です
- 細かい指定は気にせず、AIに任せましょう

**成功の確認**:
- ✅ 5つのMarkdownファイルが作成された
- ✅ 各ファイル100-200行程度で構造化されている

### ステップ3: ランディングページを3パターン作ってもらう（7分）

**AIに伝える内容**（同じチャットで続けて）:

```
【上の企画書をもとに、ランディングページを3つのデザインパターンで作ってください】

含めてほしいセクション:
- ヒーローセクション（キャッチコピー）
- 課題提起
- ソリューション紹介
- 主な機能3つ
- 料金プラン
- お問い合わせフォーム

デザインパターン:
1. モダンでシンプル（ブルー系）
2. ビジネス・プロフェッショナル（ダーク＆ゴールド）
3. カラフルで元気な感じ

HTMLとCSSで作って、かなりデザインに凝ったものにしてください。
ファイルは lp/pattern-a/, lp/pattern-b/, lp/pattern-c/ に分けてください。
```

**ポイント**:
- **6つのファイルが一気に作られる**のを見てください（これがIDE型AIの威力）
- デザインの違いを楽しんでください
- 時間があればブラウザでHTMLを開いて確認

**成功の確認**:
- ✅ `lp/` ディレクトリと3つのパターンフォルダができた
- ✅ 各フォルダに index.html と style.css がある（合計6ファイル）

### ステップ4: NotebookLMでスライド作成（6分）

**ツール**: NotebookLM（https://notebooklm.google.com/）

**手順**:

1. **NotebookLMを開く**
   - ブラウザで NotebookLM にアクセス
   - 新しいノートブックを作成

2. **Markdownファイルをドラッグ&ドロップ**
   - 先ほど作成した5つの企画書ファイルをすべて追加
   - ソースとして読み込まれる

3. **AIにスライド作成を指示**
   ```
   この企画書をもとに、顧客向けのプレゼンスライドを作ってください。

   20枚くらいで:
   - 表紙
   - 課題提起
   - ソリューション紹介
   - 主な機能
   - 導入効果
   - 料金プラン
   - FAQ
   - お問い合わせ

   わかりやすく、親しみやすいトーンで。
   ```

4. **スライド出力**
   - NotebookLMがスライド形式で生成
   - Google Slidesにエクスポート可能

**ポイント**:
- **NotebookLMは複数ファイルをソースとして読み込める**
- 企画書5ファイル全体のコンテクストからスライドが作られる
- Google Slidesで編集・プレゼン可能

**成功の確認**:
- ✅ NotebookLMでスライドが生成された
- ✅ 内容が企画書と一貫している

## 早く終わった方への追加課題

### 追加課題1: NotebookLMで投資家向けピッチ資料

**手順**:

   ```
   投資家向けのピッチ資料を作ってください。

   15枚で:
   - ビジョン
   - 課題と解決策

   - ビジネスモデル
   - 競合優位性
   - ロードマップ

   ```

## サポート体制

### 講師の巡回

- 演習中、講師が会場を巡回してサポートします
- エラーが出て止まっている方はすぐに手を挙げてください
- 質問はいつでもOKです

### よくあるエラーと対処法

#### エラー1: ファイルが生成されない
**対処法**: プロンプトを再投稿、または「上記のプロンプトで〇〇ファイルを作成してください」と明示

#### エラー2: 生成内容が期待と違う
**対処法**: 「〇〇の部分を、もっと〇〇な内容に修正してください」と具体的に指示

#### エラー3: ツールが動かない
**対処法**: 講師に相談（ブラウザ再起動、ツール再起動など）

演習終了時に、以下が達成されていればOKです:

- ✅ 企画書が5ファイルに分かれて生成された
- ✅ LPが3パターンで作成された（合計6ファイル）
- ✅ NotebookLMでスライドが作成できた
- ✅ 「ファイルが次々作られる」体験ができた

**完璧である必要はありません**。今日の目的は「体験すること」です。
