# 事前準備 - スタートガイド

## 🎯 このドキュメントの目的

VibeCoder育成プログラムを始めるための事前準備を、**3つのステップ**に分けて効率的に進められるようにしました。

**重要**: 各ドキュメントは常に最新のツールとサービスを使用することを前提としています。特定のバージョン番号は記載していないため、メンテナンスフリーで最新の環境を構築できます。

---

## 📚 事前準備の3ステップ構成

### ステップ1: アカウント作成【最優先・必須】

**ドキュメント**: [1-0-1_事前準備_アカウント作成【最優先】.md](./1-0-1_事前準備_アカウント作成【最優先】.md)

**誰が実行**: あなた（受講者）が手動で実行

**内容**:
- GitHub
- Cursor Pro
- ChatGPT Plus
- v0.dev
- Vercel
- Supabase
- GenSpark

**所要時間**: 約30〜45分

**注意**: このステップを完了してから、次のステップに進んでください。

---

### ステップ2: 開発環境構築（macOS）

**ドキュメント**: [1-0-2_事前準備_macOS完全版.md](./1-0-2_事前準備_macOS完全版.md)

**誰が実行**: **ハイブリッド実行**（ステップ0=あなた、ステップ1=Cursor）

**対象**: macOSユーザー（Apple Silicon / Intel対応）

**内容**:
- **ステップ0（手動・約5分）**: Xcode Command Line Tools + Homebrew
- **ステップ1（Cursor自動・約10分）**: Node.js, pnpm, Git, Vercel CLI, Supabase CLI, Next.jsプロジェクト

**所要時間**: 約15〜20分（手動5分 + Cursor自動10分）

**使い方**:
1. ステップ1（アカウント作成）を完了
2. Cursorアプリをインストール
3. **ステップ0**: ターミナルでXcode Command Line ToolsとHomebrewを手動インストール
4. **ステップ1**: 残りをCursor Composerに任せる

**なぜ2ステップ？**:
- Homebrewはパスワード入力が必要（セキュリティ上、Cursorからは実行不可）
- Xcode Command Line Toolsはポップアップでボタンクリックが必要

### ステップ3: 開発環境構築（Windows）

**ドキュメント**: [1-0-3_事前準備_Windows完全版.md](./1-0-3_事前準備_Windows完全版.md)

**誰が実行**: **ハイブリッド実行**（ステップ0=あなた、ステップ1=Cursor）

**対象**: Windowsユーザー（Windows 10/11）

**内容**:
- **ステップ0（手動・約3分）**: PowerShell実行ポリシー設定
- **ステップ1（Cursor自動・約12分）**: winget, Node.js, pnpm, Git, Vercel CLI, Supabase CLI, Next.jsプロジェクト

**所要時間**: 約15〜20分（手動3分 + Cursor自動12分）

**使い方**:
1. ステップ1（アカウント作成）を完了
2. Cursorアプリをインストール
3. **ステップ0**: PowerShellを管理者として開き、実行ポリシーを設定
4. **ステップ1**: 残りをCursor Composerに任せる

**なぜ2ステップ？**:
- PowerShell実行ポリシー設定は**管理者権限**が必要（セキュリティ上、Cursorからは実行不可）
- この設定を行わないと、npm/pnpm/Vercel CLI等が正常に動作しない

**Windows特有の注意事項**:
- PowerShell実行ポリシーは**必ず最初に設定**してください
- 管理者PowerShellでの操作が必要です
- 詳細な手順はドキュメント内で超丁寧に説明されています

## 💡 新しいドキュメント構成の特徴

### 1. アカウント作成を最優先化
環境構築の前に、必要なすべてのアカウントを作成することで、スムーズに開発を開始できます。

### 2. OS別の完全分離
macOSとWindowsで異なる手順を完全に分離し、混乱を防ぎます。

### 3. ハイブリッド実行形式
1-0-2と1-0-3は、**ユーザーが最低限の手動作業**（macOS: Homebrew、Windows: PowerShell設定）を行った後、**Cursorが残りを自動実行**する形式になっています。

**なぜハイブリッド？**:
- Homebrewやパスワード入力、管理者権限が必要な作業はCursorから実行不可
- しかし、それ以外はすべてCursorが自動実行できる
- 初心者でもコピペだけで手動部分を完了できるよう、超丁寧に説明

### 4. バージョンレス設計
特定のバージョン番号を記載せず、常に最新のツールを使用することを前提としています。これにより、ドキュメントのメンテナンスが不要になり、常に最新の環境を構築できます。

### 必ず順番通りに進めてください

1. **まずステップ1**: アカウント作成を完了
2. **次にステップ2または3**: お使いのOSに応じた環境構築

### Cursor Composerの使い方

各ドキュメント（1-0-2, 1-0-3）には、Cursor Composerでの実行方法が詳しく記載されています。

基本的な流れ：
1. Cursorアプリを起動
2. **Cmd + I**（macOS）または **Ctrl + I**（Windows）でComposerを開く
3. ドキュメント全体をコピー
4. Composerに貼り付け
5. Cursorが自動的にすべてをセットアップ
