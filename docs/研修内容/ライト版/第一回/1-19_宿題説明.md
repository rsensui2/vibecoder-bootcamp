# 宿題説明

## 概要
第一回講義で学習した内容を実践し、次回講義への準備を行うための宿題です。v0を使って自分なりのWebアプリUIを作成することで、AI駆動開発の基礎スキルを定着させ、実際の開発体験を積むことが目的です。

## 学習目標
- v0を使った実践的なUI作成スキルを習得する
- 学習した概念を実際のプロジェクトで応用する
- 次回講義に向けた準備を完了する
- 自分なりのアイデアを形にする体験を得る

## 内容

### 1. 宿題の概要

**課題内容：**
v0を使って、自分が実際に使いたいWebアプリケーションのUIを一つ作成してください。

**制約条件：**
- v0を使用してUI作成
- テーマは自由（ただし実用的なもの）
- 完成度より学習体験を重視
- 次回講義で共有できる状態

**期待する成果物：**
- 動作するUIプロトタイプ
- 作成過程の記録
- 学習した内容のまとめ
- 次回への質問・課題

### 3. 作成手順

**ステップ1：アイデアの整理（30分）**
```
考慮ポイント：
- 自分が本当に使いたいものか？
- 解決したい課題は明確か？
- 実現可能な範囲か？
- 他の人にも価値があるか？
```

**ステップ2：機能の洗い出し（30分）**
```
基本機能の定義：
- 必須機能（3-5個）
- あると良い機能（2-3個）
- 将来追加したい機能（1-2個）

例：家計簿アプリの場合
必須：収入・支出入力、カテゴリ分類、月次集計
あると良い：グラフ表示、予算設定
将来：レシート読み取り、銀行連携
```

**ステップ3：UI設計（1時間）**
```
画面構成の検討：
- メイン画面の構成
- 必要な画面数
- 画面間の遷移
- 重要な機能の配置

デザインの方向性：
- ターゲットユーザー
- 使用シーン
- デザインテイスト
- 色の方向性
```

**ステップ4：v0での実装（2-3時間）**
```
段階的な作成：
1. 基本レイアウトの作成
2. 主要機能の実装
3. 詳細な調整
4. レスポンシブ対応
5. 最終的な仕上げ
```

### 4. プロンプト作成のガイドライン

**効果的なプロンプトの構造：**
```
「[アプリ名]の[画面名]を作成してください。

【基本情報】
- 目的：[何を解決するアプリか]
- ユーザー：[誰が使うか]
- 使用場面：[いつ、どこで使うか]

【機能要件】
- [機能1の詳細]
- [機能2の詳細]
- [機能3の詳細]

【デザイン要件】
- 全体的なテイスト：[モダン、シンプル等]
- 色の方向性：[メインカラー、アクセント]
- レイアウト：[カード形式、リスト形式等]
- 特別な要望：[アニメーション、特殊効果等]

【技術要件】
- レスポンシブデザイン対応
- アクセシビリティ考慮
- [その他の技術的要求]」
```

**具体例：読書記録アプリ**
```
「読書記録アプリのメイン画面を作成してください。

【基本情報】
- 目的：読んだ本の記録と管理
- ユーザー：読書好きの社会人
- 使用場面：通勤中、寝る前、本屋で

【機能要件】
- 読書中の本一覧表示
- 読了本の一覧表示
- 新しい本の追加フォーム
- 読書進捗の表示（ページ数/全ページ）
- 評価とメモ機能
- 月間読書統計

【デザイン要件】
- 全体的なテイスト：温かみのあるモダン
- 色の方向性：ベージュ系、アクセントに深緑
- レイアウト：カード形式で本の表紙を重視
- 特別な要望：読書進捗バーにアニメーション

【技術要件】
- レスポンシブデザイン対応
- ダークモード対応
- タッチ操作最適化」
```

### 6. 提出方法と評価基準

**提出物：**
1. v0で作成したUIのスクリーンショット
2. 生成されたコード（可能であれば）
3. 作成過程の記録
4. 学習レポート（A4 1-2枚程度）

**評価基準：**
- **完成度**：機能の実装度合い（30%）
- **創造性**：アイデアの独創性（25%）
- **学習姿勢**：記録の詳細さ（25%）
- **実用性**：実際に使いたくなるか（20%）

**提出期限：**
次回講義の3日前まで

**提出方法：**
- メールまたは指定のプラットフォーム
- ファイル形式：PDF、画像、テキスト

### 7. よくある質問と回答

**Q: テーマが思い浮かばない場合は？**
A: 日常の小さな不便を解決するアプリを考えてみてください。「こんなアプリがあったらいいな」という体験から始めましょう。

**Q: v0の無料枠で足りない場合は？**
A: 基本的な機能に絞って作成してください。完璧を求めず、学習体験を重視しましょう。

**Q: 生成されたUIが期待と違う場合は？**
A: プロンプトをより具体的に修正して再試行してください。その過程も学習の一部です。

**Q: 技術的な部分がわからない場合は？**
A: 現段階では理解できなくても問題ありません。UIの見た目と使いやすさに集中してください。

**宿題完了後の準備：**

**持参物：**

**予習内容：**
- Cursorの基本操作動画視聴（推奨）

- 次回講義資料の事前確認（配布時）

### 9. サポート体制

**質問・相談窓口：**
- メール：[講師のメールアドレス]
- Discord：[コミュニティチャンネル]
- オフィスアワー：[指定時間]

**よくある問題の解決法：**
- v0にアクセスできない → VPN設定の確認
- プロンプトが通らない → より簡潔な表現に変更
- 期待と違う結果 → 段階的に詳細化

**緊急時の対応：**
技術的な問題で進められない場合は、早めに連絡してください。代替手段を提案します。

## 重要ポイント
- **完璧を求めない**：学習体験を重視し、完成度より過程を大切に
- **記録を残す**：成功も失敗も記録して次回の学習に活用
- **実用性を意識**：自分が本当に使いたいものを作る
- **楽しむ**：AI駆動開発の可能性を楽しみながら体験する
