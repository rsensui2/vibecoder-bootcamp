# Phase 1: 既存資料の詳細分析

**目的**: 9月版・7月版・1-1を比較分析し、統合計画を立てる

---

## ステップ1-1: 9月版をベースとして読み込む（最優先）

### 目的

9月版が内容のベースとなるため、まず9月版を完全に理解する。

### 対象ファイル

```yaml
target_file: "docs/研修内容/2025年9月/第X回/X-X_タイトル.md"
```

### 分析ポイント

#### 1. 構造の把握

```yaml
structure_analysis:
  chapter_structure:
    question: "現在の章構成は？"
    items_to_check:
      - "章数"
      - "各章のタイトル"
      - "章の順序"

  section_structure:
    question: "セクション構成は？"
    items_to_check:
      - "各章のセクション数"
      - "セクションのタイトル"
      - "セクションの論理的つながり"

  format:
    question: "Marp形式（---区切り）か？"
    items_to_check:
      - "--- で区切られているか"
      - "スライド形式か"
      - "Markdown形式か"
```

#### 2. 新規加筆内容の特定（最重要）

```yaml
new_additions:
  question: "9月版で新たに加筆された部分は何か？"
  importance: "⭐⭐⭐⭐⭐ 最優先（100%保持）"

  what_to_find:
    new_business_cases:
      - "7月版にない新しいビジネス事例"
      - "2025年の最新事例"

    new_diagrams:
      - "7月版にない新しい図解"
      - "新しいMermaid図"

    new_explanations:
      - "7月版にない新しい説明"
      - "新しい概念の解説"

    new_concepts:
      - "7月版にない新しい概念"
      - "新しい用語"

  how_to_identify:
    method_1: "7月版と比較して、存在しない部分をリストアップ"
    method_2: "9月版の特徴的な表現・事例を抽出"
    method_3: "2025年の日付・最新情報を特定"

  output_format: |
    ## 9月版の新規加筆内容（100%保持）

    ### 新しいビジネス事例
    - [事例1]
    - [事例2]

    ### 新しい図解
    - [図解1の説明]
    - [図解2の説明]

    ### 新しい説明
    - [説明1のトピック]
    - [説明2のトピック]

    ### 新しい概念
    - [概念1]
    - [概念2]
```

#### 3. 薄くなった部分の特定

```yaml
simplified_parts:
  question: "9月版で削られた・薄くなった部分は何か？"
  note: "あえて薄くした部分は7月版から補完する"

  what_to_find:
    removed_examples:
      - "7月版にあった具体例が削除"
      - "削除された理由を推測"

    simplified_explanations:
      - "7月版より簡略化された説明"
      - "詳細が削られた部分"

    removed_visuals:
      - "削除された図解"
      - "削除された表"

  how_to_identify:
    method: "7月版と比較して、9月版で不足している部分"

  output_format: |
    ## 9月版で薄くなった部分（7月版から補完）

    ### 削られた具体例
    - [例1のトピック]
    - [例2のトピック]

    ### 簡略化された説明
    - [説明1のトピック]
    - [説明2のトピック]

    ### 削除された図解・表
    - [図解1の説明]
    - [表1の説明]
```

## ステップ1-2: 7月版の充実した内容を特定

### 目的

9月版より充実している内容を取り込む。

### 対象ファイル

```yaml
target_files: "docs/研修内容/2025年7月/第X回/*.md"
note: "7月版は章ごとにファイルが分割されている可能性がある"
```

### 分析ポイント

#### 1. より充実している説明

```yaml
better_explanations:
  question: "9月版より充実している説明は何か？"

  what_to_find:
    detailed_technical_explanations:
      - "より詳しい技術解説"
      - "ステップバイステップの詳細"

    richer_examples:
      - "より豊富な具体例"
      - "複数のバリエーション"

    detailed_workflows:
      - "より詳細なワークフロー"
      - "詳細な手順"

  how_to_compare:
    method: "同じトピックを9月版と比較"
    criteria:
      - "文字数"
      - "具体例の数"
      - "ステップの詳細度"

  output_format: |
    ## 7月版の充実していた説明

    ### より詳しい技術解説
    - [トピック1]: 9月版（XX文字） → 7月版（YY文字）
    - [トピック2]: 9月版（XX文字） → 7月版（YY文字）

    ### より豊富な具体例
    - [トピック1]: 9月版（X例） → 7月版（Y例）
    - [トピック2]: 9月版（X例） → 7月版（Y例）

    ### より詳細なワークフロー
    - [ワークフロー1]: 9月版（Xステップ） → 7月版（Yステップ）
```

#### 2. 削除された重要な内容

```yaml
removed_content:
  question: "9月版で削られた重要な内容は何か？"

  what_to_find:
    removed_examples:
      - "削除された具体例"
      - "削除されたビジネス事例"

    removed_diagrams:
      - "削除された図解"
      - "削除されたフローチャート"

    removed_tips:
      - "削除されたTips"
      - "削除されたベストプラクティス"

  how_to_identify:
    method: "7月版に存在し、9月版に存在しない内容"

  output_format: |
    ## 9月版で削られた重要な内容

    ### 削除された具体例
    - [例1]: [説明]
    - [例2]: [説明]

    ### 削除された図解
    - [図解1]: [説明]
    - [図解2]: [説明]

    ### 削除されたTips
    - [Tips1]: [説明]
    - [Tips2]: [説明]
```

#### 3. 有用な図解の特定

```yaml
useful_visuals:
  question: "7月版の図解で有用なものは何か？"

  what_to_find:
    mermaid_diagrams:
      - "フローチャート"
      - "マインドマップ"
      - "比較図"

    tables:
      - "比較表"
      - "スキルマトリックス"
      - "ワークフロー表"

    other_visuals:
      - "スクリーンショット（言及）"
      - "イメージ図"

  how_to_evaluate:
    criteria:
      - "理解を助けるか"
      - "視覚的にわかりやすいか"
      - "情報量が適切か"

  output_format: |
    ## 7月版の有用な図解

    ### Mermaid図
    - [図解1]: [タイプ]、[目的]、[評価]
    - [図解2]: [タイプ]、[目的]、[評価]

    ### 表
    - [表1]: [タイプ]、[目的]、[評価]
    - [表2]: [タイプ]、[目的]、[評価]
```

## ステップ1-3: お手本（1-1）から構造・トーン・スタイルを学ぶ

### 目的

構造・トーン・視覚スタイルのお手本として1-1を参照。

### 参照ファイル

```yaml
reference_file: "docs/研修内容/2025年9月/第一回/1-1_オリエンテーションとマインドセット変革.md"
```

### 学ぶこと

#### 1. 構造

```yaml
structure:
  yaml_frontmatter:
    what: "YAMLフロントマター（7項目完備）"
    items:
      - "title"
      - "track"
      - "author"
      - "last_updated"
      - "category"
      - "duration"
      - "target_audience"

  mandatory_sections:
    what: "必須セクション（🎯📌💡🚀）の配置"
    lecture_level:
      - "## 🎯 この講義で学ぶこと"
      - "## 📌 この講義の位置づけ"
    chapter_level:
      - "## 🎯 この章で学ぶこと"
      - "## 📌 この章の位置づけ"
      - "## 💡 この章のまとめ"
      - "## 🚀 次の章への橋渡し"

  chapter_separators:
    what: "章区切り（---）の使い方"
    usage: "章と章の間に --- を挿入"

  logical_connections:
    what: "セクション間の論理的つながり"
    pattern: "前章振り返り → 本章内容 → 次章予告"
```

#### 2. トーン

```yaml
tone:
  semi_formal:
    what: "セミフォーマル"
    characteristics:
      - "敬語と平易な表現のバランス"
      - "「です・ます」調"
      - "親しみやすさを保つ"

  positive_empowerment:
    what: "ポジティブ・エンパワーメント"
    characteristics:
      - "「できる」「実現できる」を強調"
      - "課題ではなく機会として表現"
      - "自信を持たせる表現"

  non_engineer_friendly:
    what: "非エンジニア向けの平易な表現"
    characteristics:
      - "専門用語には説明を付ける"
      - "比喩を多用"
      - "日常的な言葉で表現"
```

#### 3. 視覚スタイル

```yaml
visual_style:
  mermaid_color_scheme:
    what: "Mermaid図のカラースキーム"
    colors:
      primary: "#3b82f6"
      success: "#10b981"
      warning: "#f59e0b"
      error: "#ef4444"
      other: "#8b5cf6, #fbbf24"

  table_style:
    what: "表の見せ方"
    characteristics:
      - "見出し行を太字"
      - "重要なセルを太字またはコード"
      - "3-5列が基本"

  emoji_usage:
    what: "絵文字の使い方"
    types: "🎯📌💡🚀✅"
    usage:
      - "🎯: 学習目標"
      - "📌: 位置づけ"
      - "💡: まとめ"
      - "🚀: 次への橋渡し"
      - "✅: チェックリスト"

  emphasis_techniques:
    what: "強調技法（太字、コード）"
    usage:
      - "**太字**: 重要な概念、キーワード"
      - "`コード`: ツール名、技術用語"
      - "> 引用: 重要な注意事項"
```

### 学ばないこと

```yaml
what_not_to_copy:
  content:
    what: "❌ 内容そのもの"
    reason: "内容は9月版+7月版から取る"

  word_count:
    what: "❌ 文字数"
    reason: "1-1は特殊に長い（30,000-40,000字）"
    note: "各章は柔軟に対応（500行以上）"
```
