# 講義資料生成 Skill

**作成日**: 2025年11月8日
**バージョン**: 2.0.0
**目的**: VibeCoder育成プログラムの講義資料を、9月版ベース+7月版統合+1-1構造で高品質に生成する

---

## 🎯 基本スタンス（重要）

### 対象受講者
- **レベル**: 非エンジニア
- **前提**: コーディング経験なし
- **目標**: Vibe Coder（AIにコードを書かせる人）になること

### コンテンツ生成ポリシー

#### ✅ 必ず含めるもの
- **Cursorへの指示文を記載**（「〜を作成してください」という形式）
- **何を作りたいかの説明**（目的と期待する動作）
- **AIへの依頼の仕方**（効果的なプロンプトの例）
- **概念的な理解**（なぜそうするのか）
- **ビジュアルでの説明**（図解、フロー、比較表）

#### ❌ 絶対に含めないもの
- **具体的なコード実装**（TypeScript、JavaScript等のコード）
- **プログラミング構文の詳細**（for文、if文等の説明）
- **技術的な実装手順**（手動でのコーディング作業）

#### 📝 記述例

**❌ 悪い例**（コードを書かせる）:
```typescript
function fetchUsers() {
  return fetch('/api/users')
    .then(res => res.json())
}
```

**✅ 良い例**（Cursorへの指示）:
> **Cursorへの指示**:
> 「`/api/users`エンドポイントからユーザー一覧を取得する関数を作成してください。エラーハンドリングも含めてください。」

## 📋 このSkillについて

### 何をするか（What）

9月版の講義資料（Marp形式）をベースに、7月版の充実した内容を取り込み、
1-1の構造・トーン・スタイルを適用した、11月版（Markdown形式）を生成します。

**核心的な特徴**:
- **9月版の新規加筆を100%保持**（最優先）
- **7月版の充実した内容を統合**（高優先）
- **1-1の構造・トーン・視覚スタイルを適用**（お手本）
- **Markdown形式に統一**（Marpから変換）
- **構造の明確化**（🎯📌💡🚀セクション必須）

### いつ使うか（When）

以下の条件が**すべて**揃った時に使用：

- ✅ 9月版と7月版の講義資料が両方存在する
- ✅ Markdown形式に変換したい
- ✅ 構造（🎯📌💡🚀）を追加したい
- ✅ 9月版の新規加筆を100%保持しつつ、7月版の充実した内容を統合したい
- ✅ 1-1のような高品質な講義資料を作りたい

### 使用対象

- **content-generator** Sub-agent（推奨）
- メインClaude（講義資料生成タスク時）

## 🎯 優先順位（november_2025_update_prompt.md準拠）

### 統合の優先順位

| 優先度 | 内容 | アクション | 重要度 |
|--------|------|-----------|--------|
| **1. 最優先** | 9月版の新規加筆内容 | **100%保持** | ⭐⭐⭐⭐⭐ |
| **2. 高** | 7月版の充実した内容 | **取り込む** | ⭐⭐⭐⭐ |
| **3. 中** | 1-1の構造・トーン・スタイル | **適用** | ⭐⭐⭐ |
| **4. 低** | 重複する内容 | **削除** | - |
| **5. 避ける** | 古い情報（2024年以前） | **削除** | - |

### 9月版の新規加筆（最優先）

```yaml
what_to_preserve:
  - "7月版にない新しいビジネス事例"
  - "7月版にない新しい図解"
  - "7月版にない新しい説明"
  - "7月版にない新しい概念"

action: "一文字も削らずに100%保持"
```

### 7月版の充実した内容（高優先）

```yaml
what_to_integrate:
  better_explanations:
    - "9月版より充実している説明"
    - "より詳しい技術解説"
    - "より豊富な具体例"

  removed_content:
    - "9月版で削られた重要な具体例"
    - "9月版で削られた有用な図解"
    - "9月版で削られた詳細説明"

  practical_elements:
    - "実践的なTips"
    - "よくあるエラーと対処法"
    - "ベストプラクティス"

action: "判断して取り込む"
note: "あえて薄くした部分は無理に詳しくしない"
```

### 1-1の構造・トーン・スタイル（中優先）

```yaml
reference_file: "docs/研修内容/2025年9月/第一回/1-1_オリエンテーションとマインドセット変革.md"

what_to_learn:
  structure:
    - "YAMLフロントマター（7項目完備）"
    - "必須セクション（🎯📌💡🚀）の配置"
    - "章区切り（---）の使い方"
    - "セクション間の論理的つながり"

  tone:
    - "セミフォーマル"
    - "ポジティブ・エンパワーメント"
    - "非エンジニア向けの平易な表現"

  visual_style:
    - "Mermaid図のカラースキーム"
    - "表の見せ方"
    - "絵文字の使い方"
    - "強調技法（太字、コード）"

what_not_to_copy:
  - "❌ 内容そのもの（内容は9月版+7月版から）"
  - "❌ 文字数（1-1は特殊に長い）"
```

## 📖 生成プロセス（5フェーズ）

### Phase 1: 既存資料の詳細分析

**目的**: 9月版・7月版・1-1を比較分析し、統合計画を立てる

**主要ステップ**:
1. 9月版をベースとして読み込む（最優先）
2. 7月版の充実した内容を特定
3. お手本（1-1）から構造・トーン・スタイルを学ぶ
4. 統合計画の作成

詳細: [processes/phase-1-analysis.md](processes/phase-1-analysis.md)

---

### Phase 2: 統合戦略の立案

**目的**: 優先順位・加筆基準・構造設計を決定

**主要ステップ**:
1. 統合の優先順位を確認
2. 加筆基準を適用
3. 構造設計の原則に従う

詳細: [processes/phase-2-strategy.md](processes/phase-2-strategy.md)

---

### Phase 3: コンテンツ生成

**目的**: YAMLフロントマター、必須セクション、章ごとの内容を生成

**主要ステップ**:
1. YAMLフロントマター作成（7項目完備）
2. 講義レベル必須セクション作成（🎯📌）
3. 章ごとの生成ループ（🎯📌💡🚀）
4. Mermaid図の挿入（1-3図/章）
5. 表の挿入（2-4表/章）

詳細: [processes/phase-3-generation.md](processes/phase-3-generation.md)

---

### Phase 4: 品質検証

**目的**: 統合チェック、構造チェック、定量的検証、品質基準チェック

**主要ステップ**:
1. 統合チェック（9月版100%保持、7月版統合）
2. 構造チェック（フロントマター、必須セクション）
3. 定量的検証（最低500行、図表数）
4. 品質基準チェック（lecture-quality-standards準拠）

詳細: [processes/phase-4-verification.md](processes/phase-4-verification.md)

---

### Phase 5: 反復改善

**目的**: 品質スコア45/50以上まで改善

**主要ステップ**:
1. prompt-optimization Skillによる改善
2. 改善の記録

詳細: [processes/phase-5-improvement.md](processes/phase-5-improvement.md)

## 🌟 ベストプラクティス

### 1. 段階的生成

✅ **推奨**:
```
第1章生成 → 検証 → 改善
第2章生成 → 検証 → 改善
...
全体検証 → 最終調整
```

❌ **非推奨**:
```
一気に全章を生成 → 検証 → 大幅修正が必要
```

### 2. 具体例の優先配置

✅ **推奨**:
```
理論説明 → 例1 → 理論説明 → 例2 → 例3
```

❌ **非推奨**:
```
理論説明 → 理論説明 → 理論説明 → 例1つ
```

### 3. 視覚要素の戦略的配置

✅ **推奨**:
```
テキスト → Mermaid図 → テキスト → 表 → テキスト → Mermaid図
```

❌ **非推奨**:
```
テキストのみ6ページ → Mermaid図3つ連続
```

## ⚠️ 重要な注意事項

### DO（やること）

- ✅ 9月版の新規加筆内容は**100%保持**
- ✅ 各章の冒頭に**明確な目次・学習目標**を追加
- ✅ 7月版の**充実した説明・具体例**を取り込む
- ✅ 論理的な**流れを整理**
- ✅ **Markdown形式**で作成（Marpではない）
- ✅ 1-1の**構造・トーン・スタイル**を適用

### DON'T（やらないこと）

- ❌ 9月版の新規加筆を削除・変更しない
- ❌ あえて薄くした部分を勝手に詳しくしない
- ❌ 重複する内容を繰り返さない
- ❌ Marp形式（`---`で区切ったスライド形式）にしない
- ❌ 7月版の内容を無批判にすべてコピーしない
- ❌ 1-1の内容をコピーしない（構造のみ参考）

## 📞 トラブルシューティング

### 問題: Skillが選択されない

**原因**: "what + when"が不明瞭

**解決策**:
```markdown
lecture-generator Skillを使って、以下のファイルを生成してください
```

### 問題: 9月版の新規加筆が保持されていない

**原因**: 統合時に削除されている

**解決策**:
- Phase 4のステップ4-1（統合チェック）を必ず実施
- 9月版と11月版を比較
- 不足分を即座に追加

### 問題: 定量的指標が範囲外

**原因**: 章のボリュームが不足

**解決策**:
- 最低500行以上を確保
- 7月版の充実した内容を追加統合
- Mermaid図・表を追加
