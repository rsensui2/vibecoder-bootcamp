# 構造基準詳細

VibeCoder育成プログラム講義資料の構造に関する詳細な品質基準

---

## YAMLフロントマター

### 必須項目（全7項目）

```yaml
---
title: 講義タイトル（例: 第一回：AI駆動開発入門とマインドセット変革）
track: ライト版（固定）
author: TEKION Group（固定）
last_updated: YYYY-MM-DD（生成日）
category: AI駆動開発基礎 / Webアプリケーション開発 / デプロイメント（選択）
duration: 90分（固定）
target_audience: 非エンジニア向けAI駆動開発学習者（固定）
---
```

### 項目説明

| 項目 | 説明 | 例 |
|------|------|-----|
| **title** | 講義番号 + 講義タイトル | `第一回：AI駆動開発入門とマインドセット変革` |
| **track** | トラック名（常に「ライト版」） | `ライト版` |
| **author** | 著者（常に「TEKION Group」） | `TEKION Group` |
| **last_updated** | 最終更新日（YYYY-MM-DD形式） | `2025-11-08` |
| **category** | カテゴリ（3種類から選択） | `AI駆動開発基礎` |
| **duration** | 講義時間（常に90分） | `90分` |
| **target_audience** | 対象者（固定） | `非エンジニア向けAI駆動開発学習者` |

## 見出し階層ルール

### レベル1（#）

**用途**: 講義タイトル + 各章タイトル

```markdown
# 第一回：AI駆動開発入門とマインドセット変革

（講義レベルの必須セクション）

---

# 第1章：Vibe Coderとは何か

（章レベルの必須セクション）

---

# 第2章：AIツールの全体像

（章レベルの必須セクション）
```

### レベル2（##）

**用途**: セクションタイトル（絵文字付き）

```markdown
## 🎯 この章で学ぶこと
## 📌 この章の位置づけ
## 💡 この章のまとめ
## 🚀 次の章への橋渡し
```

### レベル3（###）

**用途**: サブセクション

```markdown
### Vibe Coderの定義
### チャット型AIツールの特徴
### ワークフロー例
```

### ネスト制限

- 最大3階層まで（#、##、###）
- ####（レベル4）は使用しない
- 必要な場合は箇条書きで代用

---

## 必須セクション

### 講義レベル（ファイル冒頭）

講義資料の冒頭に必ず以下の2セクションを配置：

```markdown
# 第一回：AI駆動開発入門とマインドセット変革

## 🎯 この講義で学ぶこと

- 学習目標1
- 学習目標2
- 学習目標3
- 学習目標4

## 📌 この講義の位置づけ

この講義は、VibeCoder育成プログラムの...
```

### 章レベル（各章）

各章に必ず以下の4セクションを配置：

#### 1. 🎯 この章で学ぶこと（章の冒頭）

```markdown
## 🎯 この章で学ぶこと

- **具体的な学習目標1**
- **具体的な学習目標2**
- **具体的な学習目標3**
```

**ルール**:
- 3-6項目
- 箇条書き
- キーワードを**太字**
- 行動動詞使用（理解する、習得する、実践する）

#### 2. 📌 この章の位置づけ（学習目標の後）

```markdown
## 📌 この章の位置づけ

この章では、前章で学んだ〜を基に、〜を学びます。
次章の〜につながる重要な基礎となります。
```

**ルール**:
- 2-4段落（各3-7行）
- 前章・次章との関係を明示
- カリキュラム全体での位置づけを説明

#### 3. 💡 この章のまとめ（章の終盤）

```markdown
## 💡 この章のまとめ

- **キーポイント1**: 説明文
- **キーポイント2**: 説明文
- **キーポイント3**: 説明文
```

**ルール**:
- 3-6項目
- 箇条書き
- キーワードを**太字**
- 実践的なポイント

#### 4. 🚀 次の章への橋渡し（章の最後）

```markdown
## 🚀 次の章への橋渡し

この章では〜を学びました。次章では〜を学びます。
```

**ルール**:
- 1-2段落（各3-5行）
- 学習内容の振り返り
- 次章への期待感を醸成

## 章区切り

### 区切り記号

章と章の間には必ず以下の区切りを挿入：

```markdown
---
```

```markdown
## 💡 この章のまとめ
```

## 構造の良い例 vs 悪い例

### ❌ 悪い例

```markdown
---
title: AI開発入門
---

# AI開発について

AIを使って開発します。

# AIツール

いろんなツールがあります。
```

**問題点**:
- フロントマターが不完全（7項目必要）
- 講義レベルの必須セクション欠如
- 章レベルの必須セクション欠如
- 章区切りがない
- 見出し階層が不適切

### ✅ 良い例

```markdown
---
title: 第一回：AI駆動開発入門とマインドセット変革
track: ライト版
author: TEKION Group
last_updated: 2025-11-08
category: AI駆動開発基礎
duration: 90分
target_audience: 非エンジニア向けAI駆動開発学習者
---

# 第一回：AI駆動開発入門とマインドセット変革

## 🎯 この講義で学ぶこと

- **Vibe Coderの概念**と従来の開発手法との違い
- **AI駆動開発の全体像**とツールカテゴリ
- **実践的なワークフロー**と具体的な使い方

## 📌 この講義の位置づけ

この講義は、VibeCoder育成プログラムの第一回として、AI駆動開発の基礎概念を学びます。
後続の講義で具体的なツール（Cursor、Supabase等）を学ぶ前に、全体像を理解することが重要です。
```
