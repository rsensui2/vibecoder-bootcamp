# 文章品質基準詳細

VibeCoder育成プログラム講義資料の文章品質に関する詳細な品質基準

---

## トーン設定ガイドライン

### フォーマリティレベル

**セミフォーマル**を維持：

| 要素 | 使用する | 使用しない |
|------|---------|----------|
| **語尾** | です・ます調 | だ・である調 |
| **敬語** | 基本的な敬語 | 過度な敬語 |
| **代名詞** | あなた、皆さん | 君、お前 |
| **文体** | 説明的、教育的 | 論文調、カジュアル |

### ポジティブ・エンパワーメントスタイル

#### ✅ 良い例（ポジティブ）

```markdown
AI駆動開発を習得すれば、**非エンジニアでも本格的なWebアプリを開発できます**。
学習時間は数ヶ月で、**誰でも**Vibe Coderになれる時代が到来しました。
```

#### ❌ 悪い例（ネガティブ）

```markdown
従来の開発は難しく、習得に何年もかかります。
エンジニアでない人には不可能でした。
```

### 対象読者への配慮

**非エンジニア向け**の配慮：

- 専門用語には必ず説明を付ける
- 技術的な詳細は段階的に開示
- 「難しくない」ことを強調
- 成功事例を多く紹介

**例**:
```markdown
### APIとは

API（Application Programming Interface）は、異なるソフトウェア同士が
データをやり取りするための仕組みです。

**わかりやすく言うと**:
レストランのメニューのようなものです。メニュー（API）を見て注文すれば、
キッチン（サーバー）が料理（データ）を提供してくれます。
```

## 読みやすさ最適化

### 段落長

- **目標**: 3-7行/段落
- **最小**: 2行（1行は避ける）
- **最大**: 7行（8行以上は分割）

#### ❌ 悪い例（長すぎる）

```markdown
Cursorは、AIを統合したコードエディタで、自然言語で指示するだけでコードを生成・編集できます。従来のVS Codeの機能に加えて、GPT-4を搭載したComposerモードを備えており、要件を日本語で入力するだけで、複数ファイルにわたるコードを自動生成できます。さらに、コンテクストコントロール機能により、プロジェクト全体を理解した上でコードを生成するため、整合性の高い実装が可能です。これにより、開発速度が10倍以上向上し、非エンジニアでも本格的なアプリ開発ができるようになりました。
```

**問題点**: 1段落に詰め込みすぎ、読みにくい

#### ✅ 良い例（適切な長さ）

```markdown
Cursorは、AIを統合したコードエディタで、自然言語で指示するだけで
コードを生成・編集できます。

従来のVS Codeの機能に加えて、GPT-4を搭載したComposerモードを備えており、
要件を日本語で入力するだけで、複数ファイルにわたるコードを自動生成できます。

さらに、コンテクストコントロール機能により、プロジェクト全体を理解した上で
コードを生成するため、整合性の高い実装が可能です。

**結果**: 開発速度が10倍以上向上し、非エンジニアでも本格的なアプリ開発が
できるようになりました。
```

**良い点**: 段落を分割、読みやすい

### 文長

- **目標**: 30-60文字/文
- **平均**: 45文字
- **最大**: 80文字（それ以上は分割）

#### 文長計測例

```markdown
Cursorは、AIを統合したコードエディタです。（20文字）→ 短い、OK

自然言語で指示するだけでコードを生成・編集できます。（26文字）→ 適切

要件を日本語で入力するだけで、複数ファイルにわたるコードを自動生成できます。（37文字）→ 適切

従来のVS Codeの機能に加えて、GPT-4を搭載したComposerモードを備えており、要件を日本語で入力するだけで実装が完了します。（60文字）→ 適切

Cursorは、AIを統合したコードエディタで、自然言語で指示するだけでコードを生成・編集でき、従来のVS Codeの機能に加えてGPT-4を搭載したComposerモードを備えています。（78文字）→ やや長い、分割推奨
```

### 箇条書き

- **目標**: 3-7項目/リスト
- **最小**: 3項目（2項目以下は段落に）
- **最大**: 7項目（8項目以上は分割）
- **ネスト**: 最大2階層

#### ✅ 良い例

```markdown
### Vibe Coderに必要なスキル

- **日本語力**: 要件を明確に記述する能力
- **要件定義力**: 何を作るかを明確化する能力
- **コンテクストコントロール**: AIに適切な情報を提供する能力
- **デバッグ力**: エラーを読み解き、修正指示を出す能力
```

#### ❌ 悪い例（項目が多すぎる）

```markdown
- 項目1
- 項目2
- 項目3
- 項目4
- 項目5
- 項目6
- 項目7
- 項目8
- 項目9
- 項目10
```

**解決策**: カテゴリ分けして複数のリストに分割

## 強調技法ルール

### 太字（**...**）

**用途**:
- キーワード
- 重要概念
- 数値データ
- 製品名・ツール名（初出時）

**例**:
```markdown
**Cursor**は、AIを統合したコードエディタです。
開発速度が**10倍以上**向上します。
**Vibe Coder**に必要なスキルは、日本語力、要件定義力、コンテクストコントロールです。
```

### コード（`...`）

**用途**:
- ツール名（2回目以降）
- 技術用語
- コマンド
- ファイル名
- APIキー・環境変数

**例**:
```markdown
`Cursor`の`Composerモード`を起動するには、`Cmd+I`を押します。
`.cursorrules`ファイルにプロジェクトのルールを記述します。
`NEXT_PUBLIC_SUPABASE_URL`環境変数を設定してください。
```

### 引用（> ...）

**用途**:
- インパクトのある発言
- モチベーショナルメッセージ
- 重要な警告

**例**:
```markdown
> "AI駆動開発により、誰でもアイデアを形にできる時代が到来しました。"
> — OpenAI CEO Sam Altman

> ⚠️ **重要**: APIキーは絶対に公開リポジトリにコミットしないでください。
```

### 見出し（##, ###）

**用途**:
- セクションの明確な区切り
- 階層構造の明示

**ルール**:
- レベル2（##）: 絵文字 + タイトル（太字なし）
- レベル3（###）: タイトルのみ（太字なし、絵文字なし）

**例**:
```markdown
## 🎯 この章で学ぶこと
### Cursorの基本操作
### Composerモードの使い方
```

## 視覚的分離テクニック

### 水平線（---）

**用途**: 章区切り（章と章の間のみ）

```markdown
# 第1章：Vibe Coderとは何か

（章の内容）

---

# 第2章：AIツールの全体像

（章の内容）
```

### 空行

**用途**:
- 段落間（1行）
- セクション間（1-2行）
- コードブロック前後（1行）
- 表の前後（1行）

**例**:
```markdown
段落1の内容です。

段落2の内容です。


## 新しいセクション

内容が始まります。
```

### セクション間隔

- **章間**: 水平線（---）
- **セクション間**: 空行2行
- **段落間**: 空行1行

## 文章品質の良い例 vs 悪い例

### ❌ 悪い例

```markdown
Cursorは、AIを統合したコードエディタで、自然言語で指示するだけでコードを生成・編集できます。従来のVS Codeの機能に加えて、GPT-4を搭載したComposerモードを備えており、要件を日本語で入力するだけで、複数ファイルにわたるコードを自動生成できます。さらに、コンテクストコントロール機能により、プロジェクト全体を理解した上でコードを生成するため、整合性の高い実装が可能です。これにより、開発速度が10倍以上向上し、非エンジニアでも本格的なアプリ開発ができるようになりました。Cursorを使うには、まずダウンロードしてインストールします。その後、APIキーを設定します。プロジェクトフォルダを開きます。Composerモードを起動します。要件を入力します。
```

**問題点**:
- 段落が長すぎる（1段落150文字以上）
- 文が長すぎる（80文字以上）
- 強調がない
- 視覚的分離がない

### ✅ 良い例

```markdown
### Cursorとは

**Cursor**は、AIを統合したコードエディタで、自然言語で指示するだけで
コードを生成・編集できます。

従来のVS Codeの機能に加えて、**GPT-4を搭載したComposerモード**を備えており、
要件を日本語で入力するだけで、複数ファイルにわたるコードを自動生成できます。

さらに、**コンテクストコントロール機能**により、プロジェクト全体を理解した上で
コードを生成するため、整合性の高い実装が可能です。

**結果**: 開発速度が**10倍以上**向上し、**非エンジニアでも本格的なアプリ開発**が
できるようになりました。

### Cursorの使い方

**基本ワークフロー**:

1. Cursorをダウンロード・インストール
2. APIキーを設定
3. プロジェクトフォルダを開く
4. `Cmd+I`でComposerモードを起動
5. 要件を日本語で入力
6. コードが自動生成される

詳細は[Cursor公式ドキュメント](https://cursor.sh/docs)を参照してください。
```

**良い点**:
- 段落が適切（3-7行）
- 文が適切（30-60文字）
- キーワードが**太字**
- ツール名・コマンドが`コード`
- 視覚的に分離（見出し、空行、箇条書き）

## 専門用語の説明ルール

### 初出時のルール

専門用語は**初出時に必ず説明**を付ける：

```markdown
### APIとは

**API**（Application Programming Interface）は、異なるソフトウェア同士が
データをやり取りするための仕組みです。

**わかりやすく言うと**:
レストランのメニューのようなものです。メニュー（API）を見て注文すれば、
キッチン（サーバー）が料理（データ）を提供してくれます。
```

### 2回目以降

2回目以降は略語のみでOK：

```markdown
`API`を使って外部サービスと連携します。
`API`キーを設定すれば、すぐに利用可能です。
```

### 非エンジニア向け配慮

技術用語は**比喩・例え**を活用：

| 技術用語 | 比喩・例え |
|---------|----------|
| **API** | レストランのメニュー |
| **データベース** | Excel表の超高性能版 |
| **認証** | 会員証の提示 |
| **デプロイ** | お店のオープン |
| **フレームワーク** | 家の設計図・土台 |
