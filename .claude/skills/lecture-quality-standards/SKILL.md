# VibeCoder育成プログラム 講義資料品質基準 Skill

**作成日**: 2025年11月8日
**バージョン**: 2.0.0
**目的**: VibeCoder育成プログラムの講義資料を一貫した高品質で生成・検証する

---

## 🎯 基本スタンス（重要）

### 対象受講者
- **レベル**: 非エンジニア
- **前提**: コーディング経験なし
- **目標**: Vibe Coder（AIにコードを書かせる人）になること

### コンテンツポリシー

#### ✅ DO（やること）
- **Cursorへの指示文を記載**（「〜を作成してください」という形式）
- **何を作りたいかの説明**（目的と期待する動作）
- **AIへの依頼の仕方**（効果的なプロンプトの例）
- **概念的な理解**（なぜそうするのか）
- **ビジュアルでの説明**（図解、フロー、比較表）

#### ❌ DON'T（やらないこと）
- **具体的なコード実装**（TypeScript、JavaScript等のコード）
- **プログラミング構文の詳細**（for文、if文等の説明）
- **技術的な実装手順**（手動でのコーディング作業）

#### 📝 記述例

**❌ 悪い例**（コードを書かせる）:
```typescript
function fetchUsers() {
  return fetch('/api/users')
    .then(res => res.json())
}
```

**✅ 良い例**（Cursorへの指示）:
> **Cursorへの指示**:
> 「`/api/users`エンドポイントからユーザー一覧を取得する関数を作成してください。エラーハンドリングも含めてください。」

## 📋 このSkillについて

### 何をするか（What）

このSkillは、**非エンジニア向け**のVibeCoder育成プログラムの講義資料（1-1〜4-4）を生成・検証する際に、**96/100点（⭐⭐⭐⭐⭐）の品質基準**を保証します。

具体的には：
- 構造の一貫性（YAMLフロントマター、必須セクション、見出し階層）
- 視覚要素の最適化（絵文字、Mermaid図、表）
- 教育効果の最大化（学習目標、具体例、段階的設計）
- 文章品質の保証（トーン、読みやすさ、強調技法）
- 用語・フォーマットの統一
- **非エンジニア向けコンテンツ**（コード不要、Cursor指示中心）

### いつ使うか（When）

以下の状況で使用してください：

- **講義資料を新規作成する時**
  - content-generatorサブエージェントが講義資料を生成する
  - メインClaudeが講義資料を作成する

- **既存講義資料を更新する時**
  - 9月版を11月版にアップデートする
  - 7月版の内容を統合する
  - 構造を最新基準に合わせる

- **講義資料の品質を検証する時**
  - quality-analyzerサブエージェントが品質分析を実行する
  - consistency-checkerサブエージェントが一貫性を確認する
  - 生成後の自己チェックを行う

### 使用対象

- **content-generator** サブエージェント（生成時）
- **quality-analyzer** サブエージェント（検証時）
- **consistency-checker** サブエージェント（一貫性チェック時）
- **メインClaude**（講義資料作成時）

## 🎯 使用方法

### ステップ1: 品質基準の確認

講義資料を生成・検証する前に、以下の5つの基準カテゴリを確認してください：

1. **構造基準**: [structure-standards.md](standards/structure-standards.md)
   - YAMLフロントマター（7項目）
   - 必須セクション（🎯📌💡🚀）
   - 見出し階層

2. **視覚要素基準**: [visual-standards.md](standards/visual-standards.md)
   - 絵文字使用ルール
   - Mermaid図（1-3個/章）
   - 表フォーマット

3. **教育効果基準**: [educational-standards.md](standards/educational-standards.md)
   - 学習目標（3-6項目）
   - 具体例（理論1:実例3）
   - 段階的設計

4. **文章品質基準**: [writing-standards.md](standards/writing-standards.md)
   - トーン（セミフォーマル、ポジティブ）
   - 読みやすさ（段落3-7行、文45文字平均）
   - 強調技法

5. **一貫性基準**: [consistency-standards.md](standards/consistency-standards.md)
   - 用語統一（Vibe Coder、AI駆動開発）
   - フォーマット統一
   - トーンの一貫性

### ステップ2: 定量的目標の確認

講義資料全体で以下の定量的目標を満たしてください：

| 指標 | 目標値 | 許容範囲 |
|------|--------|---------|
| **総文字数/講義** | 34,000字 | 30,000-40,000字 |
| **章数** | 7-8章 | 6-9章 |
| **Mermaid図** | 16図/講義 | 12-20図 |
| **表** | 22表/講義 | 15-25表 |
| **理論:実例比率** | 1:3 | 1:2〜1:4 |

詳細は [定量的品質指標](#📊-定量的品質指標) を参照してください。

### ステップ3: 品質チェックリストの実行

生成・検証後、[品質チェックリスト](#✅-品質チェックリスト)を使用して以下を確認：

- ✅ 構造チェック（5項目）
- ✅ 視覚要素チェック（5項目）
- ✅ 教育効果チェック（5項目）
- ✅ 文章品質チェック（5項目）
- ✅ 一貫性チェック（5項目）

## 📊 定量的品質指標

### 文書統計目標

| 指標 | 目標値 | 許容範囲 |
|------|--------|---------|
| **総文字数/講義** | 34,000字 | 30,000-40,000字 |
| **章数** | 7-8章 | 6-9章 |
| **平均章長** | 4,250字/章 | 4,000-5,000字/章 |
| **Mermaid図** | 16図/講義 | 12-20図 |
| **図/章比率** | 2図/章 | 1.5-3図/章 |
| **表** | 22表/講義 | 15-25表 |
| **表/章比率** | 2.75表/章 | 2-4表/章 |
| **絵文字** | 120回/講義 | 80-150回 |

### コンテンツ密度目標

| 要素 | 目標比率 | 許容範囲 |
|------|----------|---------|
| **理論 vs 実例** | 1:3 | 1:2〜1:4 |
| **テキスト vs 図表** | 7:3 | 6:4〜8:2 |
| **説明 vs 実践** | 6:4 | 5:5〜7:3 |

### 読了時間目標

- **読み上げ時間**: 85分（350字/分）
- **講義時間**: 90分
- **質疑・演習時間**: 5分

## ✅ 品質チェックリスト

講義資料を生成後、以下をチェックしてください：

### 構造チェック

- [ ] YAMLフロントマターが全7項目完備
- [ ] 講義レベルで「🎯 この講義で学ぶこと」「📌 この講義の位置づけ」が存在
- [ ] 全章で4つの必須セクション（🎯📌💡🚀）が存在
- [ ] 章区切りに`---`を使用
- [ ] 見出し階層が適切（#、##、###）

### 視覚要素チェック

- [ ] 絵文字が適切に使用（80-150回）
- [ ] Mermaid図が1-3個/章
- [ ] Mermaid図のスタイリングが統一（#3b82f6, #10b981等）
- [ ] 表が適切に挿入（15-25個）
- [ ] 表の見出しが太字

### 教育効果チェック

- [ ] 学習目標が3-6項目
- [ ] 行動動詞使用（理解する、習得する、実践する）
- [ ] 具体例が豊富（理論1:実例3）
- [ ] 数値データを含む
- [ ] ビジネス事例を含む

### 文章品質チェック

- [ ] トーンがセミフォーマル
- [ ] ポジティブ・エンパワーメントな表現
- [ ] 段落長が3-7行
- [ ] 文長が30-60文字（平均45文字）
- [ ] 箇条書きが3-7項目

### 一貫性チェック

- [ ] "Vibe Coder"の表記統一（大文字V、大文字C）
- [ ] "AI駆動開発"の表記統一
- [ ] "IDE型AI"の表記統一
- [ ] ツール名が英語（Cursor, Claude, ChatGPT）
- [ ] コードブロックに言語指定

## 🔧 実装例

### 例1: content-generatorサブエージェントでの使用

```markdown
lecture-quality-standards Skillを参照して、
第二回講義 2-2_技術スタックとAI要件定義演習.md を生成してください。

【品質要件】
- 総文字数: 30,000-40,000字
- 章数: 7-8章
- Mermaid図: 12-20図
- 表: 15-25個
- 理論:実例 = 1:3

【構造要件】
- YAMLフロントマター完備
- 全章で🎯📌💡🚀セクション
- 章区切りに`---`

【品質基準】
- structure-standards.md: 構造基準
- visual-standards.md: 視覚要素
- educational-standards.md: 教育効果
- writing-standards.md: 文章品質
- consistency-standards.md: 一貫性

生成後、品質チェックリストで検証してください。
```

### 例2: quality-analyzerサブエージェントでの使用

```markdown
lecture-quality-standards Skillを参照して、
1-1_オリエンテーションとマインドセット変革.md の品質を分析してください。

【分析項目】
1. 構造チェック（5項目）
2. 視覚要素チェック（5項目）
3. 教育効果チェック（5項目）
4. 文章品質チェック（5項目）
5. 一貫性チェック（5項目）

【定量分析】
- 総文字数、章数、図表数を計測
- 目標値との比較
- 改善提案

【出力形式】
- スコア: X/100点
- 合格/不合格
- 改善ポイント
```
